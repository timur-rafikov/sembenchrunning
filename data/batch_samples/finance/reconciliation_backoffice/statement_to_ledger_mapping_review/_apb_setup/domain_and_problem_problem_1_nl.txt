--- TYPE HIERARCHY ---

  Everything that is a reconciliation case or a statement line or a ledger account or a mapping rule or a statement transaction or a counterparty or a external reference or a supporting document or a coa code or a approver or a adjustment code or a exception type or a organizational role is also a object
  Everything that is a reconciliation team or a audit team is also a organizational role
  Everything that is a automated case or a manual case is also a reconciliation case

--- ACTIONS ---

  open reconciliation case a reconciliation case A
    I can only open reconciliation case a reconciliation case A if it is not the case that the case A exists and it is not the case that A is closed
    Once I open reconciliation case a reconciliation case A, it is the case that the case A exists

  assign a statement line B to a reconciliation case A
    I can only assign a statement line B to a reconciliation case A if it is the case that the case A exists and B is unassigned and A is a candidate for the case statement B
    I can only assign a statement line B to a reconciliation case A if it is not the case that the case A has an assigned statement
    Once I assign a statement line B to a reconciliation case A, it is the case that the case A is assigned to the statement line B and the case A has an assigned statement
    Once I assign a statement line B to a reconciliation case A, it is not the case anymore that B is unassigned

  release a statement line B from a reconciliation case A
    I can only release a statement line B from a reconciliation case A if it is the case that the case A is assigned to the statement line B
    I can only release a statement line B from a reconciliation case A if it is not the case that accounting is validated for A and it is not the case that escalation is recorded for A
    Once I release a statement line B from a reconciliation case A, it is the case that B is unassigned
    Once I release a statement line B from a reconciliation case A, it is not the case anymore that the case A is assigned to the statement line B and it is not the case anymore that the case A has an assigned statement and it is not the case anymore that the case approver is engaged with A and it is not the case anymore that the evidence for A is verified and it is not the case anymore that A has an escalation flag and it is not the case anymore that A has completed post validation and it is not the case anymore that A has a manual review flag

  attach an external reference B to a reconciliation case A
    I can only attach an external reference B to a reconciliation case A if it is the case that the case A exists and B is available as an external reference
    Once I attach an external reference B to a reconciliation case A, it is the case that A is linked to the external reference B
    Once I attach an external reference B to a reconciliation case A, it is not the case anymore that B is available as an external reference

  detach an external reference B from a reconciliation case A
    I can only detach an external reference B from a reconciliation case A if it is the case that A is linked to the external reference B
    Once I detach an external reference B from a reconciliation case A, it is the case that B is available as an external reference
    Once I detach an external reference B from a reconciliation case A, it is not the case anymore that A is linked to the external reference B

  engage approver for an external reference B in a reconciliation case A
    I can only engage approver for an external reference B in a reconciliation case A if it is the case that the case A exists and the case A has an assigned statement and A is linked to the external reference B
    I can only engage approver for an external reference B in a reconciliation case A if it is not the case that the case approver is engaged with A
    Once I engage approver for an external reference B in a reconciliation case A, it is the case that the case approver is engaged with A

  engage approver with a supporting document B for a reconciliation case A
    I can only engage approver with a supporting document B for a reconciliation case A if it is the case that the case A exists and the case A has an assigned statement and B is available as a supporting document
    I can only engage approver with a supporting document B for a reconciliation case A if it is not the case that the case approver is engaged with A
    Once I engage approver with a supporting document B for a reconciliation case A, it is the case that the case approver is engaged with A and A has an escalation flag
    Once I engage approver with a supporting document B for a reconciliation case A, it is not the case anymore that B is available as a supporting document

  verify evidence for a reconciliation case A with an approver C using an external reference B
    I can only verify evidence for a reconciliation case A with an approver C using an external reference B if it is the case that the case approver is engaged with A and the case A has an assigned statement and A is linked to the external reference B and C is available as an approver
    I can only verify evidence for a reconciliation case A with an approver C using an external reference B if it is not the case that the evidence for A is verified
    Once I verify evidence for a reconciliation case A with an approver C using an external reference B, it is the case that the evidence for A is verified
    Once I verify evidence for a reconciliation case A with an approver C using an external reference B, it is not the case anymore that A has an escalation flag

  apply a coa code B as evidence for a reconciliation case A
    I can only apply a coa code B as evidence for a reconciliation case A if it is the case that the case A has an assigned statement and B is applied to A
    I can only apply a coa code B as evidence for a reconciliation case A if it is not the case that the evidence for A is verified
    Once I apply a coa code B as evidence for a reconciliation case A, it is the case that the evidence for A is verified
    Once I apply a coa code B as evidence for a reconciliation case A, it is not the case anymore that A has an escalation flag

  propose a statement transaction B as a transaction candidate for a reconciliation case A
    I can only propose a statement transaction B as a transaction candidate for a reconciliation case A if it is the case that the case A exists and B is available in the statement transaction and A is eligible for the statement transaction B
    Once I propose a statement transaction B as a transaction candidate for a reconciliation case A, it is the case that B is a transaction candidate for A
    Once I propose a statement transaction B as a transaction candidate for a reconciliation case A, it is not the case anymore that B is available in the statement transaction

  revoke a statement transaction B as a transaction candidate for a reconciliation case A
    I can only revoke a statement transaction B as a transaction candidate for a reconciliation case A if it is the case that B is a transaction candidate for A
    Once I revoke a statement transaction B as a transaction candidate for a reconciliation case A, it is the case that B is available in the statement transaction
    Once I revoke a statement transaction B as a transaction candidate for a reconciliation case A, it is not the case anymore that B is a transaction candidate for A

  propose a mapping rule B as a rule candidate for a reconciliation case A
    I can only propose a mapping rule B as a rule candidate for a reconciliation case A if it is the case that the case A exists and B is available in the mapping rule and A is eligible for the mapping rule B
    Once I propose a mapping rule B as a rule candidate for a reconciliation case A, it is the case that B is a rule candidate for A
    Once I propose a mapping rule B as a rule candidate for a reconciliation case A, it is not the case anymore that B is available in the mapping rule

  revoke a mapping rule B as a rule candidate for a reconciliation case A

    Once I revoke a mapping rule B as a rule candidate for a reconciliation case A, it is not the case anymore that B is a rule candidate for A

    Once I propose a counterparty B as a candidate for a reconciliation case A, it is the case that B is a counterparty candidate for A
    Once I propose a counterparty B as a candidate for a reconciliation case A, it is not the case anymore that B is available as a counterparty

  revoke a counterparty B as a candidate for a reconciliation case A
    I can only revoke a counterparty B as a candidate for a reconciliation case A if it is the case that B is a counterparty candidate for A
    Once I revoke a counterparty B as a candidate for a reconciliation case A, it is the case that B is available as a counterparty
    Once I revoke a counterparty B as a candidate for a reconciliation case A, it is not the case anymore that B is a counterparty candidate for A

  propose an exception type B as an exception candidate for a reconciliation case A
    I can only propose an exception type B as an exception candidate for a reconciliation case A if it is the case that the case A exists and B is available as an exception type and A is eligible for an exception of type B
    Once I propose an exception type B as an exception candidate for a reconciliation case A, it is the case that B is an exception candidate for A

    Once I revoke an exception type B as an exception candidate for a reconciliation case A, it is the case that B is available as an exception type
    Once I revoke an exception type B as an exception candidate for a reconciliation case A, it is not the case anymore that B is an exception candidate for A

  validate a mapping rule D for a reconciliation case A against a ledger account B using a statement transaction C
    I can only validate a mapping rule D for a reconciliation case A against a ledger account B using a statement transaction C if it is the case that the case A has an assigned statement and C is a transaction candidate for A and D is a rule candidate for A and B is available in the ledger account and A is a candidate for the ledger account B

    Once I validate a mapping rule D for a reconciliation case A against a ledger account B using a statement transaction C, it is the case that B is validated for A and accounting is validated for A
    Once I validate a mapping rule D for a reconciliation case A against a ledger account B using a statement transaction C, it is not the case anymore that B is available in the ledger account

  validate a reconciliation case A mapping with a counterparty C and an adjustment code D for a ledger account B

    I can only validate a reconciliation case A mapping with a counterparty C and an adjustment code D for a ledger account B if it is not the case that accounting is validated for A
    Once I validate a reconciliation case A mapping with a counterparty C and an adjustment code D for a ledger account B, it is the case that B is validated for A and accounting is validated for A and A has a manual review flag and A has an escalation flag
    Once I validate a reconciliation case A mapping with a counterparty C and an adjustment code D for a ledger account B, it is not the case anymore that B is available in the ledger account and it is not the case anymore that D is available as an adjustment code

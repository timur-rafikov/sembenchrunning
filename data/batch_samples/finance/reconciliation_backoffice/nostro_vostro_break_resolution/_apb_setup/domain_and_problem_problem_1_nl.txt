--- TYPE HIERARCHY ---

  Everything that is a reconciliation case or a external transaction or a ledger entry or a account or a payment message or a fee entry or a instruction reference or a evidence document or a settlement event or a operator or a system ticket or a counterparty record or a resolution group is also a object
  Everything that is a resolver team or a escalation group is also a resolution group
  Everything that is a nostro case or a vostro case is also a reconciliation case

--- ACTIONS ---

  create a reconciliation case A if it is closed and not registered
    I can only create a reconciliation case A if it is closed and not registered if it is not the case that the case A is registered and it is not the case that A is closed
    Once I create a reconciliation case A if it is closed and not registered, it is the case that the case A is registered

  link an external transaction B to a reconciliation case A
    I can only link an external transaction B to a reconciliation case A if it is the case that the case A is registered and B is available as an external transaction and A is a candidate for the transaction B
    I can only link an external transaction B to a reconciliation case A if it is not the case that the transaction is linked to the case A
    Once I link an external transaction B to a reconciliation case A, it is the case that the case A is linked to the transaction B and the transaction is linked to the case A
    Once I link an external transaction B to a reconciliation case A, it is not the case anymore that B is available as an external transaction

  unlink an external transaction B from a reconciliation case A
    I can only unlink an external transaction B from a reconciliation case A if it is the case that the case A is linked to the transaction B
    I can only unlink an external transaction B from a reconciliation case A if it is not the case that A is accounting aligned and it is not the case that a remediation ticket is created for A
    Once I unlink an external transaction B from a reconciliation case A, it is the case that B is available as an external transaction
    Once I unlink an external transaction B from a reconciliation case A, it is not the case anymore that the case A is linked to the transaction B and it is not the case anymore that the transaction is linked to the case A and it is not the case anymore that the instruction is validated for A and it is not the case anymore that evidence is attached to A and it is not the case anymore that A is pending remediation and it is not the case anymore that A has been processed for remediation and it is not the case anymore that A requires manager approval

  attach an instruction reference B to a reconciliation case A
    I can only attach an instruction reference B to a reconciliation case A if it is the case that the case A is registered and B is available
    Once I attach an instruction reference B to a reconciliation case A, it is the case that A is a case instruction reference candidate for B
    Once I attach an instruction reference B to a reconciliation case A, it is not the case anymore that B is available

  detach an instruction reference B from a reconciliation case A
    I can only detach an instruction reference B from a reconciliation case A if it is the case that A is a case instruction reference candidate for B
    Once I detach an instruction reference B from a reconciliation case A, it is the case that B is available
    Once I detach an instruction reference B from a reconciliation case A, it is not the case anymore that A is a case instruction reference candidate for B

  validate an instruction reference B for a reconciliation case A
    I can only validate an instruction reference B for a reconciliation case A if it is the case that the case A is registered and the transaction is linked to the case A and A is a case instruction reference candidate for B
    I can only validate an instruction reference B for a reconciliation case A if it is not the case that the instruction is validated for A
    Once I validate an instruction reference B for a reconciliation case A, it is the case that the instruction is validated for A

  attach an evidence document B to a reconciliation case A and validate instruction
    I can only attach an evidence document B to a reconciliation case A and validate instruction if it is the case that the case A is registered and the transaction is linked to the case A and B is available
    I can only attach an evidence document B to a reconciliation case A and validate instruction if it is not the case that the instruction is validated for A
    Once I attach an evidence document B to a reconciliation case A and validate instruction, it is the case that the instruction is validated for A and A is pending remediation
    Once I attach an evidence document B to a reconciliation case A and validate instruction, it is not the case anymore that B is available

  submit evidence for review for a reconciliation case A with an instruction reference B by an operator C
    I can only submit evidence for review for a reconciliation case A with an instruction reference B by an operator C if it is the case that the instruction is validated for A and the transaction is linked to the case A and A is a case instruction reference candidate for B and C is available
    I can only submit evidence for review for a reconciliation case A with an instruction reference B by an operator C if it is not the case that evidence is attached to A
    Once I submit evidence for review for a reconciliation case A with an instruction reference B by an operator C, it is the case that evidence is attached to A
    Once I submit evidence for review for a reconciliation case A with an instruction reference B by an operator C, it is not the case anymore that A is pending remediation

  attach a settlement event B to a reconciliation case A
    I can only attach a settlement event B to a reconciliation case A if it is the case that the transaction is linked to the case A and A is linked to the case settlement for B
    I can only attach a settlement event B to a reconciliation case A if it is not the case that evidence is attached to A
    Once I attach a settlement event B to a reconciliation case A, it is the case that evidence is attached to A
    Once I attach a settlement event B to a reconciliation case A, it is not the case anymore that A is pending remediation

  link a payment message B to a reconciliation case A
    I can only link a payment message B to a reconciliation case A if it is the case that the case A is registered and B is available as a payment message and A is a candidate for the payment message B
    Once I link a payment message B to a reconciliation case A, it is the case that A is linked to the payment message B
    Once I link a payment message B to a reconciliation case A, it is not the case anymore that B is available as a payment message

  unlink a payment message B from a reconciliation case A
    I can only unlink a payment message B from a reconciliation case A if it is the case that A is linked to the payment message B
    Once I unlink a payment message B from a reconciliation case A, it is the case that B is available as a payment message
    Once I unlink a payment message B from a reconciliation case A, it is not the case anymore that A is linked to the payment message B

  link an account B to a reconciliation case A
    I can only link an account B to a reconciliation case A if it is the case that the case A is registered and B is available and A is a candidate for the account B
    Once I link an account B to a reconciliation case A, it is the case that A is linked to the account B
    Once I link an account B to a reconciliation case A, it is not the case anymore that B is available

  unlink an account B from a reconciliation case A
    I can only unlink an account B from a reconciliation case A if it is the case that A is linked to the account B
    Once I unlink an account B from a reconciliation case A, it is the case that B is available
    Once I unlink an account B from a reconciliation case A, it is not the case anymore that A is linked to the account B

  link a fee entry B to a reconciliation case A
    I can only link a fee entry B to a reconciliation case A if it is the case that the case A is registered and B is available and A is a candidate for the fee entry B
    Once I link a fee entry B to a reconciliation case A, it is the case that A is linked to the fee entry B
    Once I link a fee entry B to a reconciliation case A, it is not the case anymore that B is available

  unlink a fee entry B from a reconciliation case A
    I can only unlink a fee entry B from a reconciliation case A if it is the case that A is linked to the fee entry B
    Once I unlink a fee entry B from a reconciliation case A, it is the case that B is available
    Once I unlink a fee entry B from a reconciliation case A, it is not the case anymore that A is linked to the fee entry B

  link a counterparty record B to a reconciliation case A
    I can only link a counterparty record B to a reconciliation case A if it is the case that the case A is registered and B is available and A is a candidate for the counterparty record B
    Once I link a counterparty record B to a reconciliation case A, it is the case that A is linked to the counterparty record B
    Once I link a counterparty record B to a reconciliation case A, it is not the case anymore that B is available

  unlink a counterparty record B from a reconciliation case A
    I can only unlink a counterparty record B from a reconciliation case A if it is the case that A is linked to the counterparty record B
    Once I unlink a counterparty record B from a reconciliation case A, it is the case that B is available
    Once I unlink a counterparty record B from a reconciliation case A, it is not the case anymore that A is linked to the counterparty record B

  create accounting alignment candidate for a reconciliation case A with a ledger entry B, a payment message C, and an account D
    I can only create accounting alignment candidate for a reconciliation case A with a ledger entry B, a payment message C, and an account D if it is the case that the transaction is linked to the case A and A is linked to the payment message C and A is linked to the account D and B is available as a ledger entry and A is a candidate for the ledger entry B
    I can only create accounting alignment candidate for a reconciliation case A with a ledger entry B, a payment message C, and an account D if it is not the case that A is accounting aligned
    Once I create accounting alignment candidate for a reconciliation case A with a ledger entry B, a payment message C, and an account D, it is the case that A is linked to the ledger entry B and A is accounting aligned
    Once I create accounting alignment candidate for a reconciliation case A with a ledger entry B, a payment message C, and an account D, it is not the case anymore that B is available as a ledger entry

  create complex accounting alignment for a reconciliation case A with a ledger entry B, a fee entry C, and a system ticket D
    I can only create complex accounting alignment for a reconciliation case A with a ledger entry B, a fee entry C, and a system ticket D if it is the case that the transaction is linked to the case A and A is linked to the fee entry C and D is open and B is available as a ledger entry and A is a candidate for the ledger entry B and A is a candidate for the system ticket D
    I can only create complex accounting alignment for a reconciliation case A with a ledger entry B, a fee entry C, and a system ticket D if it is not the case that A is accounting aligned
    Once I create complex accounting alignment for a reconciliation case A with a ledger entry B, a fee entry C, and a system ticket D, it is the case that A is linked to the ledger entry B and A is accounting aligned and A requires manager approval and A is pending remediation
    Once I create complex accounting alignment for a reconciliation case A with a ledger entry B, a fee entry C, and a system ticket D, it is not the case anymore that B is available as a ledger entry and it is not the case anymore that D is open

  approve accounting alignment for a reconciliation case A with a payment message B and an account C
    I can only approve accounting alignment for a reconciliation case A with a payment message B and an account C if it is the case that A is accounting aligned and A requires manager approval and A is linked to the payment message B and A is linked to the account C
    Once I approve accounting alignment for a reconciliation case A with a payment message B and an account C, it is not the case anymore that A requires manager approval and it is not the case anymore that A is pending remediation

  create remediation ticket for a reconciliation case A linked to a payment message B and an account C assigned to a resolver team D
    I can only create remediation ticket for a reconciliation case A linked to a payment message B and an account C assigned to a resolver team D if it is the case that evidence is attached to A and A is accounting aligned and A is linked to the payment message B and A is linked to the account C and A is assigned to the resolution queue D
    I can only create remediation ticket for a reconciliation case A linked to a payment message B and an account C assigned to a resolver team D if it is not the case that A is pending remediation and it is not the case that a remediation ticket is created for A
    Once I create remediation ticket for a reconciliation case A linked to a payment message B and an account C assigned to a resolver team D, it is the case that a remediation ticket is created for A

  escalate and create remediation ticket for a reconciliation case A linked to a fee entry B and a counterparty record C assigned to an escalation group D
    I can only escalate and create remediation ticket for a reconciliation case A linked to a fee entry B and a counterparty record C assigned to an escalation group D if it is the case that evidence is attached to A and A is accounting aligned and A is linked to the fee entry B and A is linked to the counterparty record C and A is assigned to the resolution queue D
    I can only escalate and create remediation ticket for a reconciliation case A linked to a fee entry B and a counterparty record C assigned to an escalation group D if it is not the case that a remediation ticket is created for A
    Once I escalate and create remediation ticket for a reconciliation case A linked to a fee entry B and a counterparty record C assigned to an escalation group D, it is the case that a remediation ticket is created for A and A is pending remediation

  apply remediation and mark a reconciliation case A processed linked to a payment message B and an account C
    I can only apply remediation and mark a reconciliation case A processed linked to a payment message B and an account C if it is the case that a remediation ticket is created for A and A is pending remediation and A is linked to the payment message B and A is linked to the account C
    Once I apply remediation and mark a reconciliation case A processed linked to a payment message B and an account C, it is the case that A has been processed for remediation
    Once I apply remediation and mark a reconciliation case A processed linked to a payment message B and an account C, it is not the case anymore that A is pending remediation and it is not the case anymore that evidence is attached to A and it is not the case anymore that A requires manager approval

  reapply evidence after remediation for a reconciliation case A using an instruction reference B with an operator C
    I can only reapply evidence after remediation for a reconciliation case A using an instruction reference B with an operator C if it is the case that A has been processed for remediation and a remediation ticket is created for A and the transaction is linked to the case A and A is a case instruction reference candidate for B and C is available
    I can only reapply evidence after remediation for a reconciliation case A using an instruction reference B with an operator C if it is not the case that evidence is attached to A
    Once I reapply evidence after remediation for a reconciliation case A using an instruction reference B with an operator C, it is the case that evidence is attached to A

  verify an instruction reference B for settlement matching of a reconciliation case A
    I can only verify an instruction reference B for settlement matching of a reconciliation case A if it is the case that a remediation ticket is created for A and evidence is attached to A and A is a case instruction reference candidate for B
    I can only verify an instruction reference B for settlement matching of a reconciliation case A if it is not the case that A is pending remediation and it is not the case that A is ready for settlement matching
    Once I verify an instruction reference B for settlement matching of a reconciliation case A, it is the case that A is ready for settlement matching

  verify an evidence document B for settlement matching of a reconciliation case A
    I can only verify an evidence document B for settlement matching of a reconciliation case A if it is the case that a remediation ticket is created for A and evidence is attached to A and B is available
    I can only verify an evidence document B for settlement matching of a reconciliation case A if it is not the case that A is pending remediation and it is not the case that A is ready for settlement matching
    Once I verify an evidence document B for settlement matching of a reconciliation case A, it is the case that A is ready for settlement matching
    Once I verify an evidence document B for settlement matching of a reconciliation case A, it is not the case anymore that B is available

  confirm settlement association between a reconciliation case A and a settlement event B
    I can only confirm settlement association between a reconciliation case A and a settlement event B if it is the case that A is ready for settlement matching and B is available and A is a case settlement candidate from nostro for B
    Once I confirm settlement association between a reconciliation case A and a settlement event B, it is the case that A has a confirmed settlement with B
    Once I confirm settlement association between a reconciliation case A and a settlement event B, it is not the case anymore that B is available

  link cross ledger settlement event for a vostro case A and a nostro case B with a settlement event C
    I can only link cross ledger settlement event for a vostro case A and a nostro case B with a settlement event C if it is the case that the case A is registered and A is a vostro view and A is a case settlement candidate from vostro for C and B has a confirmed settlement with C
    I can only link cross ledger settlement event for a vostro case A and a nostro case B with a settlement event C if it is not the case that A is linked to the case settlement for C
    Once I link cross ledger settlement event for a vostro case A and a nostro case B with a settlement event C, it is the case that A is linked to the case settlement for C

  mark evidence review complete for a vostro view A with an instruction reference B by an operator C
    I can only mark evidence review complete for a vostro view A with an instruction reference B by an operator C if it is the case that the case A is registered and A is a vostro view and evidence is attached to A and A is ready for settlement matching and A is a case instruction reference candidate for B and C is available
    I can only mark evidence review complete for a vostro view A with an instruction reference B by an operator C if it is not the case that the operator review for A is completed
    Once I mark evidence review complete for a vostro view A with an instruction reference B by an operator C, it is the case that the operator review for A is completed

  close a nostro view A with signoff
    I can only close a nostro view A with signoff if it is the case that A is a nostro view and the case A is registered and the transaction is linked to the case A and A is accounting aligned and a remediation ticket is created for A and A is ready for settlement matching and evidence is attached to A
    I can only close a nostro view A with signoff if it is not the case that A is closed
    Once I close a nostro view A with signoff, it is the case that A is closed

  close a vostro view A with a settlement event B
    I can only close a vostro view A with a settlement event B if it is the case that A is a vostro view and the case A is registered and the transaction is linked to the case A and A is accounting aligned and a remediation ticket is created for A and A is ready for settlement matching and evidence is attached to A and A is linked to the case settlement for B
    I can only close a vostro view A with a settlement event B if it is not the case that A is closed

  close a vostro view A with operator acknowledgement

    I can only close a vostro view A with operator acknowledgement if it is not the case that A is closed

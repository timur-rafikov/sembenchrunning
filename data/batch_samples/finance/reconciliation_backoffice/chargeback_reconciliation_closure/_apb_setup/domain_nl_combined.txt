--- TYPE HIERARCHY ---

  Everything that is a entity is also a object
  Everything that is a chargeback case or a source transaction or a accounting batch or a merchant account or a clearing instruction or a dispute record or a work item or a analyst or a external reference or a approver or a evidence document or a reason code or a auxiliary tag is also a entity
  Everything that is a resolution method or a accounting policy is also a auxiliary tag
  Everything that is a operational case or a audit case is also a chargeback case

--- ACTIONS ---

  activate a chargeback case A
    I can only activate a chargeback case A if it is not the case that A is active and it is not the case that A is closed
    Once I activate a chargeback case A, it is the case that A is active

  link a source transaction B to a chargeback case A
    I can only link a source transaction B to a chargeback case A if it is the case that A is active and B is available and A is a candidate for the source transaction B
    I can only link a source transaction B to a chargeback case A if it is not the case that A has a match
    Once I link a source transaction B to a chargeback case A, it is the case that A has the source transaction B and A has a match
    Once I link a source transaction B to a chargeback case A, it is not the case anymore that B is available

  unlink a source transaction B from a chargeback case A
    I can only unlink a source transaction B from a chargeback case A if it is the case that A has the source transaction B
    I can only unlink a source transaction B from a chargeback case A if it is not the case that A is accounting aligned and it is not the case that A is resolution eligible
    Once I unlink a source transaction B from a chargeback case A, it is the case that B is available
    Once I unlink a source transaction B from a chargeback case A, it is not the case anymore that A has the source transaction B and it is not the case anymore that A has a match and it is not the case anymore that A is assigned for investigation and it is not the case anymore that the evidence for A is verified and it is not the case anymore that A is pending approval and it is not the case anymore that A is resolved and it is not the case anymore that the chargeback case A requires follow up

  assign a work item B to a chargeback case A
    I can only assign a work item B to a chargeback case A if it is the case that A is active and B is available
    Once I assign a work item B to a chargeback case A, it is the case that B is assigned to the chargeback case A
    Once I assign a work item B to a chargeback case A, it is not the case anymore that B is available

  release a work item B from a chargeback case A
    I can only release a work item B from a chargeback case A if it is the case that B is assigned to the chargeback case A
    Once I release a work item B from a chargeback case A, it is the case that B is available
    Once I release a work item B from a chargeback case A, it is not the case anymore that B is assigned to the chargeback case A

  record investigation assignment for a chargeback case A with a work item B
    I can only record investigation assignment for a chargeback case A with a work item B if it is the case that A is active and A has a match and B is assigned to the chargeback case A
    I can only record investigation assignment for a chargeback case A with a work item B if it is not the case that A is assigned for investigation
    Once I record investigation assignment for a chargeback case A with a work item B, it is the case that A is assigned for investigation

  assign an analyst B to a chargeback case A for investigation
    I can only assign an analyst B to a chargeback case A for investigation if it is the case that A is active and A has a match and B is available
    I can only assign an analyst B to a chargeback case A for investigation if it is not the case that A is assigned for investigation
    Once I assign an analyst B to a chargeback case A for investigation, it is the case that A is assigned for investigation and A is pending approval
    Once I assign an analyst B to a chargeback case A for investigation, it is not the case anymore that B is available

  record attestation from an approver C for a work item B in a chargeback case A
    I can only record attestation from an approver C for a work item B in a chargeback case A if it is the case that A is assigned for investigation and A has a match and B is assigned to the chargeback case A and C is available
    I can only record attestation from an approver C for a work item B in a chargeback case A if it is not the case that the evidence for A is verified
    Once I record attestation from an approver C for a work item B in a chargeback case A, it is the case that the evidence for A is verified
    Once I record attestation from an approver C for a work item B in a chargeback case A, it is not the case anymore that A is pending approval

  record attestation for a chargeback case A using an external reference B
    I can only record attestation for a chargeback case A using an external reference B if it is the case that A has a match and A has an external reference B that is validated
    I can only record attestation for a chargeback case A using an external reference B if it is not the case that the evidence for A is verified
    Once I record attestation for a chargeback case A using an external reference B, it is the case that the evidence for A is verified
    Once I record attestation for a chargeback case A using an external reference B, it is not the case anymore that A is pending approval

  link a clearing instruction B to a chargeback case A
    I can only link a clearing instruction B to a chargeback case A if it is the case that A is active and B is available and A is a candidate for the clearing instruction B
    Once I link a clearing instruction B to a chargeback case A, it is the case that A has the clearing instruction B
    Once I link a clearing instruction B to a chargeback case A, it is not the case anymore that B is available

  unlink a clearing instruction B from a chargeback case A
    I can only unlink a clearing instruction B from a chargeback case A if it is the case that A has the clearing instruction B
    Once I unlink a clearing instruction B from a chargeback case A, it is the case that B is available
    Once I unlink a clearing instruction B from a chargeback case A, it is not the case anymore that A has the clearing instruction B

  link a merchant account B to a chargeback case A

    Once I unlink a merchant account B from a chargeback case A, it is the case that B is available
    Once I unlink a merchant account B from a chargeback case A, it is not the case anymore that A has the merchant account B

  link a dispute record B to a chargeback case A
    I can only link a dispute record B to a chargeback case A if it is the case that A is active and B is available and A is a candidate for the dispute record B
    Once I link a dispute record B to a chargeback case A, it is the case that A has the dispute record B
    Once I link a dispute record B to a chargeback case A, it is not the case anymore that B is available

  unlink a dispute record B from a chargeback case A
    I can only unlink a dispute record B from a chargeback case A if it is the case that A has the dispute record B
    Once I unlink a dispute record B from a chargeback case A, it is the case that B is available
    Once I unlink a dispute record B from a chargeback case A, it is not the case anymore that A has the dispute record B

  link a reason code B to a chargeback case A
    I can only link a reason code B to a chargeback case A if it is the case that A is active and B is available and A is a candidate for the reason code B
    Once I link a reason code B to a chargeback case A, it is the case that A has the reason code B
    Once I link a reason code B to a chargeback case A, it is not the case anymore that B is available

  unlink a reason code B from a chargeback case A
    I can only unlink a reason code B from a chargeback case A if it is the case that A has the reason code B
    Once I unlink a reason code B from a chargeback case A, it is the case that B is available
    Once I unlink a reason code B from a chargeback case A, it is not the case anymore that A has the reason code B

  validate accounting alignment of a chargeback case A with an accounting batch B using a clearing instruction C and a merchant account D
    I can only validate accounting alignment of a chargeback case A with an accounting batch B using a clearing instruction C and a merchant account D if it is the case that A has a match and A has the clearing instruction C and A has the merchant account D and B is available and A is a candidate for the accounting batch B
    I can only validate accounting alignment of a chargeback case A with an accounting batch B using a clearing instruction C and a merchant account D if it is not the case that A is accounting aligned
    Once I validate accounting alignment of a chargeback case A with an accounting batch B using a clearing instruction C and a merchant account D, it is the case that A has the accounting batch B and A is accounting aligned
    Once I validate accounting alignment of a chargeback case A with an accounting batch B using a clearing instruction C and a merchant account D, it is not the case anymore that B is available

  validate accounting of a chargeback case A with an accounting batch B, a dispute record C, and an evidence document D
    I can only validate accounting of a chargeback case A with an accounting batch B, a dispute record C, and an evidence document D if it is the case that A has a match and A has the dispute record C and D is available and B is available and A is a candidate for the accounting batch B and A is a candidate for the evidence document D
    I can only validate accounting of a chargeback case A with an accounting batch B, a dispute record C, and an evidence document D if it is not the case that A is accounting aligned
    Once I validate accounting of a chargeback case A with an accounting batch B, a dispute record C, and an evidence document D, it is the case that A has the accounting batch B and A is accounting aligned and the chargeback case A requires follow up and A is pending approval
    Once I validate accounting of a chargeback case A with an accounting batch B, a dispute record C, and an evidence document D, it is not the case anymore that B is available and it is not the case anymore that D is available

  confirm accounting for a chargeback case A using a clearing instruction B and a merchant account C and clear follow up
    I can only confirm accounting for a chargeback case A using a clearing instruction B and a merchant account C and clear follow up if it is the case that A is accounting aligned and the chargeback case A requires follow up and A has the clearing instruction B and A has the merchant account C
    Once I confirm accounting for a chargeback case A using a clearing instruction B and a merchant account C and clear follow up, it is not the case anymore that the chargeback case A requires follow up and it is not the case anymore that A is pending approval

  apply a resolution method D to a chargeback case A using a clearing instruction B and a merchant account C
    I can only apply a resolution method D to a chargeback case A using a clearing instruction B and a merchant account C if it is the case that the evidence for A is verified and A is accounting aligned and A has the clearing instruction B and A has the merchant account C and A has the auxiliary tag D
    I can only apply a resolution method D to a chargeback case A using a clearing instruction B and a merchant account C if it is not the case that A is pending approval and it is not the case that A is resolution eligible
    Once I apply a resolution method D to a chargeback case A using a clearing instruction B and a merchant account C, it is the case that A is resolution eligible

  apply resolution with an accounting policy D to a chargeback case A using a dispute record B and a reason code C
    I can only apply resolution with an accounting policy D to a chargeback case A using a dispute record B and a reason code C if it is the case that the evidence for A is verified and A is accounting aligned and A has the dispute record B and A has the reason code C and A has the auxiliary tag D
    I can only apply resolution with an accounting policy D to a chargeback case A using a dispute record B and a reason code C if it is not the case that A is resolution eligible
    Once I apply resolution with an accounting policy D to a chargeback case A using a dispute record B and a reason code C, it is the case that A is resolution eligible and A is pending approval

  finalize resolution for a chargeback case A using a clearing instruction B and a merchant account C
    I can only finalize resolution for a chargeback case A using a clearing instruction B and a merchant account C if it is the case that A is resolution eligible and A is pending approval and A has the clearing instruction B and A has the merchant account C
    Once I finalize resolution for a chargeback case A using a clearing instruction B and a merchant account C, it is the case that A is resolved
    Once I finalize resolution for a chargeback case A using a clearing instruction B and a merchant account C, it is not the case anymore that A is pending approval and it is not the case anymore that the evidence for A is verified and it is not the case anymore that the chargeback case A requires follow up

  collect evidence for a chargeback case A using a work item B with an approver C
    I can only collect evidence for a chargeback case A using a work item B with an approver C if it is the case that A is resolved and A is resolution eligible and A has a match and B is assigned to the chargeback case A and C is available
    I can only collect evidence for a chargeback case A using a work item B with an approver C if it is not the case that the evidence for A is verified
    Once I collect evidence for a chargeback case A using a work item B with an approver C, it is the case that the evidence for A is verified

  record approver assignment for a chargeback case A using a work item B

    I can only record approver assignment for a chargeback case A using a work item B if it is not the case that A is pending approval and it is not the case that A has an approver recorded

  record approver assignment for a chargeback case A by an analyst B

    Once I record approver assignment for a chargeback case A by an analyst B, it is not the case anymore that B is available

  associate an external reference B with a chargeback case A
    I can only associate an external reference B with a chargeback case A if it is the case that A has an approver recorded and B is available and the chargeback case A allows an external reference link B
    Once I associate an external reference B with a chargeback case A, it is the case that A has the external reference candidate B
    Once I associate an external reference B with a chargeback case A, it is not the case anymore that B is available

  cross reference an external reference C for closure between an audit case A and an operational case B
    I can only cross reference an external reference C for closure between an audit case A and an operational case B if it is the case that A is active and A has an audit routing flag and A has an external reference C and B has the external reference candidate C
    I can only cross reference an external reference C for closure between an audit case A and an operational case B if it is not the case that A has an external reference C that is validated
    Once I cross reference an external reference C for closure between an audit case A and an operational case B, it is the case that A has an external reference C that is validated

  record audit signoff for a chargeback case A with a work item B by an approver C
    I can only record audit signoff for a chargeback case A with a work item B by an approver C if it is the case that A is active and A has an audit routing flag and the evidence for A is verified and A has an approver recorded and B is assigned to the chargeback case A and C is available
    I can only record audit signoff for a chargeback case A with a work item B by an approver C if it is not the case that A has a recorded audit signoff
    Once I record audit signoff for a chargeback case A with a work item B by an approver C, it is the case that A has a recorded audit signoff

  close a chargeback case A operationally
    I can only close a chargeback case A operationally if it is the case that A has an operational routing flag and A is active and A has a match and A is accounting aligned and A is resolution eligible and A has an approver recorded and the evidence for A is verified
    I can only close a chargeback case A operationally if it is not the case that A is closed
    Once I close a chargeback case A operationally, it is the case that A is closed

  close a chargeback case A with an external reference B
    I can only close a chargeback case A with an external reference B if it is the case that A has an audit routing flag and A is active and A has a match and A is accounting aligned and A is resolution eligible and A has an approver recorded and the evidence for A is verified and A has an external reference B that is validated
    I can only close a chargeback case A with an external reference B if it is not the case that A is closed
    Once I close a chargeback case A with an external reference B, it is the case that A is closed

    I can only close a chargeback case A with audit signoff if it is not the case that A is closed
    Once I close a chargeback case A with audit signoff, it is the case that A is closed

--- ACTIONS ---

  register a reconciliation case A
    I can only register a reconciliation case A if it is not the case that the case A is registered and it is not the case that A is closed
    Once I register a reconciliation case A, it is the case that the case A is registered

  link a ledger transaction B to a reconciliation case A
    I can only link a ledger transaction B to a reconciliation case A if it is the case that the case A is registered and B is available and A is a candidate for the transaction B
    I can only link a ledger transaction B to a reconciliation case A if it is not the case that the case A has matches
    Once I link a ledger transaction B to a reconciliation case A, it is the case that the case A is linked to the transaction B and the case A has matches
    Once I link a ledger transaction B to a reconciliation case A, it is not the case anymore that B is available

  unlink a ledger transaction B from a reconciliation case A
    I can only unlink a ledger transaction B from a reconciliation case A if it is the case that the case A is linked to the transaction B
    I can only unlink a ledger transaction B from a reconciliation case A if it is not the case that A is verified by accounting and it is not the case that A has an escalation requested
    Once I unlink a ledger transaction B from a reconciliation case A, it is the case that B is available
    Once I unlink a ledger transaction B from a reconciliation case A, it is not the case anymore that the case A is linked to the transaction B and it is not the case anymore that the case A has matches and it is not the case anymore that the investigation for A has started and it is not the case anymore that the validation for A is complete and it is not the case anymore that A is pending escalation and it is not the case anymore that A has been approved for escalation and it is not the case anymore that A requires manual review

  attach a source file B to a reconciliation case A
    I can only attach a source file B to a reconciliation case A if it is the case that the case A is registered and B is available
    Once I attach a source file B to a reconciliation case A, it is the case that A has the source file B attached
    Once I attach a source file B to a reconciliation case A, it is not the case anymore that B is available

  detach a source file B from a reconciliation case A
    I can only detach a source file B from a reconciliation case A if it is the case that A has the source file B attached
    Once I detach a source file B from a reconciliation case A, it is the case that B is available
    Once I detach a source file B from a reconciliation case A, it is not the case anymore that A has the source file B attached

  initiate investigation for a reconciliation case A with a source file B
    I can only initiate investigation for a reconciliation case A with a source file B if it is the case that the case A is registered and the case A has matches and A has the source file B attached
    I can only initiate investigation for a reconciliation case A with a source file B if it is not the case that the investigation for A has started
    Once I initiate investigation for a reconciliation case A with a source file B, it is the case that the investigation for A has started

  assign a reconciliation operator B to a reconciliation case A
    I can only assign a reconciliation operator B to a reconciliation case A if it is the case that the case A is registered and the case A has matches and B is available
    I can only assign a reconciliation operator B to a reconciliation case A if it is not the case that the investigation for A has started
    Once I assign a reconciliation operator B to a reconciliation case A, it is the case that the investigation for A has started and A is pending escalation
    Once I assign a reconciliation operator B to a reconciliation case A, it is not the case anymore that B is available

  ingest a document reference C from a source file B into a reconciliation case A
    I can only ingest a document reference C from a source file B into a reconciliation case A if it is the case that the investigation for A has started and the case A has matches and A has the source file B attached and C is available
    I can only ingest a document reference C from a source file B into a reconciliation case A if it is not the case that the validation for A is complete
    Once I ingest a document reference C from a source file B into a reconciliation case A, it is the case that the validation for A is complete
    Once I ingest a document reference C from a source file B into a reconciliation case A, it is not the case anymore that A is pending escalation

  apply a validation profile B to a reconciliation case A
    I can only apply a validation profile B to a reconciliation case A if it is the case that the case A has matches and B is applied to the case A
    I can only apply a validation profile B to a reconciliation case A if it is not the case that the validation for A is complete
    Once I apply a validation profile B to a reconciliation case A, it is the case that the validation for A is complete
    Once I apply a validation profile B to a reconciliation case A, it is not the case anymore that A is pending escalation

  link a matching key B to a reconciliation case A
    I can only link a matching key B to a reconciliation case A if it is the case that the case A is registered and B is available and A is a candidate for the matching key B
    Once I link a matching key B to a reconciliation case A, it is the case that A is linked to the matching key B
    Once I link a matching key B to a reconciliation case A, it is not the case anymore that B is available

  unlink a matching key B from a reconciliation case A

    Once I unlink a matching key B from a reconciliation case A, it is the case that B is available
    Once I unlink a matching key B from a reconciliation case A, it is not the case anymore that A is linked to the matching key B

  run accounting alignment check for a reconciliation case A with a settlement instruction B, a matching key C, and a counterparty account D
    I can only run accounting alignment check for a reconciliation case A with a settlement instruction B, a matching key C, and a counterparty account D if it is the case that the case A has matches and A is linked to the matching key C and A is linked to the counterparty account D and B is available and A is a candidate for the settlement B
    I can only run accounting alignment check for a reconciliation case A with a settlement instruction B, a matching key C, and a counterparty account D if it is not the case that A is verified by accounting
    Once I run accounting alignment check for a reconciliation case A with a settlement instruction B, a matching key C, and a counterparty account D, it is the case that A is linked to the settlement instruction B and A is verified by accounting
    Once I run accounting alignment check for a reconciliation case A with a settlement instruction B, a matching key C, and a counterparty account D, it is not the case anymore that B is available

  run settlement reconciliation with confirmation for a reconciliation case A using a settlement instruction B, a payment instruction C, and a counterparty confirmation D
    I can only run settlement reconciliation with confirmation for a reconciliation case A using a settlement instruction B, a payment instruction C, and a counterparty confirmation D if it is the case that the case A has matches and A is linked to the payment instruction C and D is available and B is available and A is a candidate for the settlement B and A is a candidate for the counterparty confirmation D

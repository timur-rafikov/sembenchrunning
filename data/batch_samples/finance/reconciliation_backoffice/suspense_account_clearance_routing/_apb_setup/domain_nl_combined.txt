DOMAIN IN NATURAL LANGUAGE
============================================================

--- TYPE HIERARCHY ---

  Everything that is a reconciliation case or a ledger account or a clearance instruction or a source system or a remediation activity or a escalation option or a suspense bucket or a communication reference or a supporting document or a authorized verifier or a audit ticket or a exception reason code or a organizational unit is also a object
  Everything that is a resolver unit or a approver unit is also a organizational unit
  Everything that is a client initiated case or a system initiated case is also a reconciliation case

--- ACTIONS ---

  open a reconciliation case A
    I can only open a reconciliation case A if it is not the case that the case A is registered and it is not the case that A is closed
    Once I open a reconciliation case A, it is the case that the case A is registered

  assign a ledger account B to a reconciliation case A
    I can only assign a ledger account B to a reconciliation case A if it is the case that the case A is registered and B is available and A is a candidate for the account B
    I can only assign a ledger account B to a reconciliation case A if it is not the case that the account for case A is claimed
    Once I assign a ledger account B to a reconciliation case A, it is the case that the case A is linked to the ledger account B and the account for case A is claimed
    Once I assign a ledger account B to a reconciliation case A, it is not the case anymore that B is available

  unassign a ledger account B from a reconciliation case A
    I can only unassign a ledger account B from a reconciliation case A if it is the case that the case A is linked to the ledger account B
    I can only unassign a ledger account B from a reconciliation case A if it is not the case that case A has executed clearance flag and it is not the case that case approval for A is recorded
    Once I unassign a ledger account B from a reconciliation case A, it is the case that B is available
    Once I unassign a ledger account B from a reconciliation case A, it is not the case anymore that the case A is linked to the ledger account B and it is not the case anymore that the account for case A is claimed and it is not the case anymore that the initial validation for case A has passed and it is not the case anymore that the evidence for case A has been verified and it is not the case anymore that A requires approval and it is not the case anymore that A has a committed resolution and it is not the case anymore that A requires follow up

  assign a reconciliation case A to a suspense bucket B
    I can only assign a reconciliation case A to a suspense bucket B if it is the case that the case A is registered and B is available
    Once I assign a reconciliation case A to a suspense bucket B, it is the case that A is assigned to the suspense bucket B
    Once I assign a reconciliation case A to a suspense bucket B, it is not the case anymore that B is available

  release a reconciliation case A from a suspense bucket B
    I can only release a reconciliation case A from a suspense bucket B if it is the case that A is assigned to the suspense bucket B
    Once I release a reconciliation case A from a suspense bucket B, it is the case that B is available
    Once I release a reconciliation case A from a suspense bucket B, it is not the case anymore that A is assigned to the suspense bucket B

  confirm initial validation for a reconciliation case A in a suspense bucket B
    I can only confirm initial validation for a reconciliation case A in a suspense bucket B if it is the case that the case A is registered and the account for case A is claimed and A is assigned to the suspense bucket B
    I can only confirm initial validation for a reconciliation case A in a suspense bucket B if it is not the case that the initial validation for case A has passed
    Once I confirm initial validation for a reconciliation case A in a suspense bucket B, it is the case that the initial validation for case A has passed

  record communication verification for a reconciliation case A using a communication reference B
    I can only record communication verification for a reconciliation case A using a communication reference B if it is the case that the case A is registered and the account for case A is claimed and B is available
    I can only record communication verification for a reconciliation case A using a communication reference B if it is not the case that the initial validation for case A has passed
    Once I record communication verification for a reconciliation case A using a communication reference B, it is the case that the initial validation for case A has passed and A requires approval
    Once I record communication verification for a reconciliation case A using a communication reference B, it is not the case anymore that B is available

  verify evidence for a reconciliation case A in a suspense bucket B by an authorized verifier C
    I can only verify evidence for a reconciliation case A in a suspense bucket B by an authorized verifier C if it is the case that the initial validation for case A has passed and the account for case A is claimed and A is assigned to the suspense bucket B and C is available
    I can only verify evidence for a reconciliation case A in a suspense bucket B by an authorized verifier C if it is not the case that the evidence for case A has been verified
    Once I verify evidence for a reconciliation case A in a suspense bucket B by an authorized verifier C, it is the case that the evidence for case A has been verified
    Once I verify evidence for a reconciliation case A in a suspense bucket B by an authorized verifier C, it is not the case anymore that A requires approval

  verify evidence for a reconciliation case A using a supporting document B
    I can only verify evidence for a reconciliation case A using a supporting document B if it is the case that the account for case A is claimed and B is linked to A
    I can only verify evidence for a reconciliation case A using a supporting document B if it is not the case that the evidence for case A has been verified
    Once I verify evidence for a reconciliation case A using a supporting document B, it is the case that the evidence for case A has been verified
    Once I verify evidence for a reconciliation case A using a supporting document B, it is not the case anymore that A requires approval

  assign a remediation activity B to a reconciliation case A
    I can only assign a remediation activity B to a reconciliation case A if it is the case that the case A is registered and B is available and A is a candidate for the remediation activity B
    Once I assign a remediation activity B to a reconciliation case A, it is the case that case A has been assigned remediation activity B
    Once I assign a remediation activity B to a reconciliation case A, it is not the case anymore that B is available

  release a remediation activity B from a reconciliation case A
    I can only release a remediation activity B from a reconciliation case A if it is the case that case A has been assigned remediation activity B
    Once I release a remediation activity B from a reconciliation case A, it is the case that B is available
    Once I release a remediation activity B from a reconciliation case A, it is not the case anymore that case A has been assigned remediation activity B

  assign a source system B to a reconciliation case A
    I can only assign a source system B to a reconciliation case A if it is the case that the case A is registered and B is available and A is a candidate from the source system B
    Once I assign a source system B to a reconciliation case A, it is the case that case A has been assigned to source system B
    Once I assign a source system B to a reconciliation case A, it is not the case anymore that B is available

  release a source system B from a reconciliation case A
    I can only release a source system B from a reconciliation case A if it is the case that case A has been assigned to source system B
    Once I release a source system B from a reconciliation case A, it is the case that B is available
    Once I release a source system B from a reconciliation case A, it is not the case anymore that case A has been assigned to source system B

  assign an escalation option B to a reconciliation case A
    I can only assign an escalation option B to a reconciliation case A if it is the case that the case A is registered and B is available and A is a candidate for escalation with the option B
    Once I assign an escalation option B to a reconciliation case A, it is the case that case A has been assigned escalation option B
    Once I assign an escalation option B to a reconciliation case A, it is not the case anymore that B is available

  release an escalation option B from a reconciliation case A
    I can only release an escalation option B from a reconciliation case A if it is the case that case A has been assigned escalation option B
    Once I release an escalation option B from a reconciliation case A, it is the case that B is available
    Once I release an escalation option B from a reconciliation case A, it is not the case anymore that case A has been assigned escalation option B

  assign an exception reason code B to a reconciliation case A
    I can only assign an exception reason code B to a reconciliation case A if it is the case that the case A is registered and B is available and A is a candidate for the exception reason code B
    Once I assign an exception reason code B to a reconciliation case A, it is the case that case A has been assigned exception code B
    Once I assign an exception reason code B to a reconciliation case A, it is not the case anymore that B is available

  release an exception reason code B from a reconciliation case A
    I can only release an exception reason code B from a reconciliation case A if it is the case that case A has been assigned exception code B
    Once I release an exception reason code B from a reconciliation case A, it is the case that B is available
    Once I release an exception reason code B from a reconciliation case A, it is not the case anymore that case A has been assigned exception code B

  apply a clearance instruction B to a reconciliation case A with a remediation activity C from a source system D
    I can only apply a clearance instruction B to a reconciliation case A with a remediation activity C from a source system D if it is the case that the account for case A is claimed and case A has been assigned remediation activity C and case A has been assigned to source system D and B is available and A is a candidate for the instruction B
    I can only apply a clearance instruction B to a reconciliation case A with a remediation activity C from a source system D if it is not the case that case A has executed clearance flag
    Once I apply a clearance instruction B to a reconciliation case A with a remediation activity C from a source system D, it is the case that case A has applied clearance instruction B and case A has executed clearance flag
    Once I apply a clearance instruction B to a reconciliation case A with a remediation activity C from a source system D, it is not the case anymore that B is available

  execute a clearance instruction B with an escalation option C and an audit ticket D for a reconciliation case A
    I can only execute a clearance instruction B with an escalation option C and an audit ticket D for a reconciliation case A if it is the case that the account for case A is claimed and case A has been assigned escalation option C and D is available and B is available and A is a candidate for the instruction B and A is a candidate for an audit ticket D
    I can only execute a clearance instruction B with an escalation option C and an audit ticket D for a reconciliation case A if it is not the case that case A has executed clearance flag
    Once I execute a clearance instruction B with an escalation option C and an audit ticket D for a reconciliation case A, it is the case that case A has applied clearance instruction B and case A has executed clearance flag and A requires follow up and A requires approval
    Once I execute a clearance instruction B with an escalation option C and an audit ticket D for a reconciliation case A, it is not the case anymore that B is available and it is not the case anymore that D is available

  finalize posting and release follow up for a reconciliation case A with a remediation activity B in a source system C
    I can only finalize posting and release follow up for a reconciliation case A with a remediation activity B in a source system C if it is the case that case A has executed clearance flag and A requires follow up and case A has been assigned remediation activity B and case A has been assigned to source system C
    Once I finalize posting and release follow up for a reconciliation case A with a remediation activity B in a source system C, it is not the case anymore that A requires follow up and it is not the case anymore that A requires approval

  record approval for a reconciliation case A with a remediation activity B in a source system C by a resolver unit D
    I can only record approval for a reconciliation case A with a remediation activity B in a source system C by a resolver unit D if it is the case that the evidence for case A has been verified and case A has executed clearance flag and case A has been assigned remediation activity B and case A has been assigned to source system C and A is a candidate for the organizational unit D
    I can only record approval for a reconciliation case A with a remediation activity B in a source system C by a resolver unit D if it is not the case that A requires approval and it is not the case that case approval for A is recorded
    Once I record approval for a reconciliation case A with a remediation activity B in a source system C by a resolver unit D, it is the case that case approval for A is recorded

  record escalated approval for a reconciliation case A with an escalation option B and an exception reason code C by an approver unit D
    I can only record escalated approval for a reconciliation case A with an escalation option B and an exception reason code C by an approver unit D if it is the case that the evidence for case A has been verified and case A has executed clearance flag and case A has been assigned escalation option B and case A has been assigned exception code C and A is a candidate for the organizational unit D
    I can only record escalated approval for a reconciliation case A with an escalation option B and an exception reason code C by an approver unit D if it is not the case that case approval for A is recorded
    Once I record escalated approval for a reconciliation case A with an escalation option B and an exception reason code C by an approver unit D, it is the case that case approval for A is recorded and A requires approval

  commit resolution and clear flags for a reconciliation case A with a remediation activity B from a source system C
    I can only commit resolution and clear flags for a reconciliation case A with a remediation activity B from a source system C if it is the case that case approval for A is recorded and A requires approval and case A has been assigned remediation activity B and case A has been assigned to source system C
    Once I commit resolution and clear flags for a reconciliation case A with a remediation activity B from a source system C, it is the case that A has a committed resolution
    Once I commit resolution and clear flags for a reconciliation case A with a remediation activity B from a source system C, it is not the case anymore that A requires approval and it is not the case anymore that the evidence for case A has been verified and it is not the case anymore that A requires follow up

  reverify evidence for a reconciliation case A in a suspense bucket B with an authorized verifier C after commit
    I can only reverify evidence for a reconciliation case A in a suspense bucket B with an authorized verifier C after commit if it is the case that A has a committed resolution and case approval for A is recorded and the account for case A is claimed and A is assigned to the suspense bucket B and C is available
    I can only reverify evidence for a reconciliation case A in a suspense bucket B with an authorized verifier C after commit if it is not the case that the evidence for case A has been verified
    Once I reverify evidence for a reconciliation case A in a suspense bucket B with an authorized verifier C after commit, it is the case that the evidence for case A has been verified

  flag a reconciliation case A in a suspense bucket B as requiring supporting document
    I can only flag a reconciliation case A in a suspense bucket B as requiring supporting document if it is the case that case approval for A is recorded and the evidence for case A has been verified and A is assigned to the suspense bucket B
    I can only flag a reconciliation case A in a suspense bucket B as requiring supporting document if it is not the case that A requires approval and it is not the case that A has supporting evidence
    Once I flag a reconciliation case A in a suspense bucket B as requiring supporting document, it is the case that A has supporting evidence

  link a communication reference B to a reconciliation case A
    I can only link a communication reference B to a reconciliation case A if it is the case that case approval for A is recorded and the evidence for case A has been verified and B is available
    I can only link a communication reference B to a reconciliation case A if it is not the case that A requires approval and it is not the case that A has supporting evidence
    Once I link a communication reference B to a reconciliation case A, it is the case that A has supporting evidence
    Once I link a communication reference B to a reconciliation case A, it is not the case anymore that B is available

  assign a supporting document B to a reconciliation case A
    I can only assign a supporting document B to a reconciliation case A if it is the case that A has supporting evidence and B is available and A references the document B
    Once I assign a supporting document B to a reconciliation case A, it is the case that A is a candidate for the supporting document B
    Once I assign a supporting document B to a reconciliation case A, it is not the case anymore that B is available

  link a supporting document C across a system initiated case A and a client initiated case B
    I can only link a supporting document C across a system initiated case A and a client initiated case B if it is the case that the case A is registered and A is from the system origin and C is a candidate for association with A and B is a candidate for the supporting document C
    I can only link a supporting document C across a system initiated case A and a client initiated case B if it is not the case that C is linked to A
    Once I link a supporting document C across a system initiated case A and a client initiated case B, it is the case that C is linked to A

  synchronize supporting documents for a reconciliation case A in a suspense bucket B with an authorized verifier C
    I can only synchronize supporting documents for a reconciliation case A in a suspense bucket B with an authorized verifier C if it is the case that the case A is registered and A is from the system origin and the evidence for case A has been verified and A has supporting evidence and A is assigned to the suspense bucket B and C is available
    I can only synchronize supporting documents for a reconciliation case A in a suspense bucket B with an authorized verifier C if it is not the case that documentation for A is confirmed
    Once I synchronize supporting documents for a reconciliation case A in a suspense bucket B with an authorized verifier C, it is the case that documentation for A is confirmed

  close a reconciliation case A with standard checks
    I can only close a reconciliation case A with standard checks if it is the case that A is from the client origin and the case A is registered and the account for case A is claimed and case A has executed clearance flag and case approval for A is recorded and A has supporting evidence and the evidence for case A has been verified
    I can only close a reconciliation case A with standard checks if it is not the case that A is closed
    Once I close a reconciliation case A with standard checks, it is the case that A is closed

  close a reconciliation case A after documented clearance with a supporting document B
    I can only close a reconciliation case A after documented clearance with a supporting document B if it is the case that A is from the system origin and the case A is registered and the account for case A is claimed and case A has executed clearance flag and case approval for A is recorded and A has supporting evidence and the evidence for case A has been verified and B is linked to A
    I can only close a reconciliation case A after documented clearance with a supporting document B if it is not the case that A is closed
    Once I close a reconciliation case A after documented clearance with a supporting document B, it is the case that A is closed

  close a reconciliation case A after evidence synchronization
    I can only close a reconciliation case A after evidence synchronization if it is the case that A is from the system origin and the case A is registered and the account for case A is claimed and case A has executed clearance flag and case approval for A is recorded and A has supporting evidence and the evidence for case A has been verified and documentation for A is confirmed
    I can only close a reconciliation case A after evidence synchronization if it is not the case that A is closed
    Once I close a reconciliation case A after evidence synchronization, it is the case that A is closed

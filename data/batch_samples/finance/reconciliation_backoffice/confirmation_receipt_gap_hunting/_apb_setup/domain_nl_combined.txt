--- TYPE HIERARCHY ---

  Everything that is a reconciliation case or a ledger entry or a posting batch or a external confirmation or a trade leg or a settlement event or a evidence attachment or a third party feed item or a approval token or a operator identity or a compliance document or a counterparty reference or a index reference is also a object
  Everything that is a escalation bucket or a exception category is also a index reference
  Everything that is a related case or a parent case is also a reconciliation case

--- ACTIONS ---

  activate a reconciliation case A
    I can only activate a reconciliation case A if it is not the case that A is active and it is not the case that A is closed
    Once I activate a reconciliation case A, it is the case that A is active

  reserve a ledger entry B for a reconciliation case A
    I can only reserve a ledger entry B for a reconciliation case A if it is the case that A is active and B is available in the ledger and A is a possible ledger candidate for B
    I can only reserve a ledger entry B for a reconciliation case A if it is not the case that A has a case assignment flag
    Once I reserve a ledger entry B for a reconciliation case A, it is the case that B is assigned to A and A has a case assignment flag
    Once I reserve a ledger entry B for a reconciliation case A, it is not the case anymore that B is available in the ledger

  release reservation of a ledger entry B from a reconciliation case A
    I can only release reservation of a ledger entry B from a reconciliation case A if it is the case that B is assigned to A
    I can only release reservation of a ledger entry B from a reconciliation case A if it is not the case that A has a verification flag and it is not the case that A is ready for resolution
    Once I release reservation of a ledger entry B from a reconciliation case A, it is the case that B is available in the ledger
    Once I release reservation of a ledger entry B from a reconciliation case A, it is not the case anymore that B is assigned to A and it is not the case anymore that A has a case assignment flag and it is not the case anymore that A has verified case evidence and it is not the case anymore that A has been validated and it is not the case anymore that A has an escalation flag and it is not the case anymore that A is ready for final evidence collection and it is not the case anymore that A is under additional review flag

  attach an evidence attachment B to a reconciliation case A
    I can only attach an evidence attachment B to a reconciliation case A if it is the case that A is active and B is available as an attachment
    Once I attach an evidence attachment B to a reconciliation case A, it is the case that A is linked to the evidence attachment B
    Once I attach an evidence attachment B to a reconciliation case A, it is not the case anymore that B is available as an attachment

  detach an evidence attachment B from a reconciliation case A
    I can only detach an evidence attachment B from a reconciliation case A if it is the case that A is linked to the evidence attachment B
    Once I detach an evidence attachment B from a reconciliation case A, it is the case that B is available as an attachment
    Once I detach an evidence attachment B from a reconciliation case A, it is not the case anymore that A is linked to the evidence attachment B

  verify evidence for a reconciliation case A using an evidence attachment B
    I can only verify evidence for a reconciliation case A using an evidence attachment B if it is the case that A is active and A has a case assignment flag and A is linked to the evidence attachment B
    I can only verify evidence for a reconciliation case A using an evidence attachment B if it is not the case that A has verified case evidence
    Once I verify evidence for a reconciliation case A using an evidence attachment B, it is the case that A has verified case evidence

  verify evidence for a reconciliation case A using a third party feed item B
    I can only verify evidence for a reconciliation case A using a third party feed item B if it is the case that A is active and A has a case assignment flag and B is available in the third party feed
    I can only verify evidence for a reconciliation case A using a third party feed item B if it is not the case that A has verified case evidence
    Once I verify evidence for a reconciliation case A using a third party feed item B, it is the case that A has verified case evidence and A has an escalation flag
    Once I verify evidence for a reconciliation case A using a third party feed item B, it is not the case anymore that B is available in the third party feed

  attach evidence an evidence attachment B to a reconciliation case A by an operator identity C
    I can only attach evidence an evidence attachment B to a reconciliation case A by an operator identity C if it is the case that A has verified case evidence and A has a case assignment flag and A is linked to the evidence attachment B and C is available
    I can only attach evidence an evidence attachment B to a reconciliation case A by an operator identity C if it is not the case that A has been validated
    Once I attach evidence an evidence attachment B to a reconciliation case A by an operator identity C, it is the case that A has been validated
    Once I attach evidence an evidence attachment B to a reconciliation case A by an operator identity C, it is not the case anymore that A has an escalation flag

  validate a reconciliation case A using an approval token B
    I can only validate a reconciliation case A using an approval token B if it is the case that A has a case assignment flag and A has consumed approval token B
    I can only validate a reconciliation case A using an approval token B if it is not the case that A has been validated
    Once I validate a reconciliation case A using an approval token B, it is the case that A has been validated
    Once I validate a reconciliation case A using an approval token B, it is not the case anymore that A has an escalation flag

  reserve a trade leg B for a reconciliation case A
    I can only reserve a trade leg B for a reconciliation case A if it is the case that A is active and B is available in the trade leg and A is a possible trade leg for B
    Once I reserve a trade leg B for a reconciliation case A, it is the case that A is linked to the trade leg B
    Once I reserve a trade leg B for a reconciliation case A, it is not the case anymore that B is available in the trade leg

  release reservation of a trade leg B from a reconciliation case A
    I can only release reservation of a trade leg B from a reconciliation case A if it is the case that A is linked to the trade leg B
    Once I release reservation of a trade leg B from a reconciliation case A, it is the case that B is available in the trade leg
    Once I release reservation of a trade leg B from a reconciliation case A, it is not the case anymore that A is linked to the trade leg B

  reserve an external confirmation B for a reconciliation case A
    I can only reserve an external confirmation B for a reconciliation case A if it is the case that A is active and B is available for external confirmation and A is a possible confirmation for B
    Once I reserve an external confirmation B for a reconciliation case A, it is the case that A is linked to the external confirmation B
    Once I reserve an external confirmation B for a reconciliation case A, it is not the case anymore that B is available for external confirmation

  release reservation of an external confirmation B from a reconciliation case A
    I can only release reservation of an external confirmation B from a reconciliation case A if it is the case that A is linked to the external confirmation B
    Once I release reservation of an external confirmation B from a reconciliation case A, it is the case that B is available for external confirmation
    Once I release reservation of an external confirmation B from a reconciliation case A, it is not the case anymore that A is linked to the external confirmation B

  reserve a settlement event B for a reconciliation case A

    Once I reserve a settlement event B for a reconciliation case A, it is the case that A is linked to the settlement event B
    Once I reserve a settlement event B for a reconciliation case A, it is not the case anymore that B is available for the settlement event

  release reservation of a settlement event B from a reconciliation case A
    I can only release reservation of a settlement event B from a reconciliation case A if it is the case that A is linked to the settlement event B
    Once I release reservation of a settlement event B from a reconciliation case A, it is the case that B is available for the settlement event
    Once I release reservation of a settlement event B from a reconciliation case A, it is not the case anymore that A is linked to the settlement event B

  reserve a counterparty reference B for a reconciliation case A
    I can only reserve a counterparty reference B for a reconciliation case A if it is the case that A is active and B is available and A is a possible counterparty for B
    Once I reserve a counterparty reference B for a reconciliation case A, it is the case that A is linked to the counterparty reference B
    Once I reserve a counterparty reference B for a reconciliation case A, it is not the case anymore that B is available

  release a counterparty reference B reservation for a reconciliation case A
    I can only release a counterparty reference B reservation for a reconciliation case A if it is the case that A is linked to the counterparty reference B
    Once I release a counterparty reference B reservation for a reconciliation case A, it is the case that B is available
    Once I release a counterparty reference B reservation for a reconciliation case A, it is not the case anymore that A is linked to the counterparty reference B

  validate a reconciliation case A with a posting batch B, a trade leg C, and an external confirmation D
    I can only validate a reconciliation case A with a posting batch B, a trade leg C, and an external confirmation D if it is the case that A has a case assignment flag and A is linked to the trade leg C and A is linked to the external confirmation D and B is available in the posting batch and A is a possible posting batch for B
    I can only validate a reconciliation case A with a posting batch B, a trade leg C, and an external confirmation D if it is not the case that A has a verification flag
    Once I validate a reconciliation case A with a posting batch B, a trade leg C, and an external confirmation D, it is the case that A is linked to the posting batch B and A has a verification flag
    Once I validate a reconciliation case A with a posting batch B, a trade leg C, and an external confirmation D, it is not the case anymore that B is available in the posting batch

  validate a reconciliation case A with a posting batch B, a settlement event C, and a compliance document D
    I can only validate a reconciliation case A with a posting batch B, a settlement event C, and a compliance document D if it is the case that A has a case assignment flag and A is linked to the settlement event C and D is available and B is available in the posting batch and A is a possible posting batch for B and A is a possible compliance document for D
    I can only validate a reconciliation case A with a posting batch B, a settlement event C, and a compliance document D if it is not the case that A has a verification flag
    Once I validate a reconciliation case A with a posting batch B, a settlement event C, and a compliance document D, it is the case that A is linked to the posting batch B and A has a verification flag and A is under additional review flag and A has an escalation flag
    Once I validate a reconciliation case A with a posting batch B, a settlement event C, and a compliance document D, it is not the case anymore that B is available in the posting batch and it is not the case anymore that D is available

  clear additional review flag for a reconciliation case A linked to a trade leg B and an external confirmation C
    I can only clear additional review flag for a reconciliation case A linked to a trade leg B and an external confirmation C if it is the case that A has a verification flag and A is under additional review flag and A is linked to the trade leg B and A is linked to the external confirmation C
    Once I clear additional review flag for a reconciliation case A linked to a trade leg B and an external confirmation C, it is not the case anymore that A is under additional review flag and it is not the case anymore that A has an escalation flag

  mark a reconciliation case A as ready for resolution linked to a trade leg B, an external confirmation C, and an escalation bucket D
    I can only mark a reconciliation case A as ready for resolution linked to a trade leg B, an external confirmation C, and an escalation bucket D if it is the case that A has been validated and A has a verification flag and A is linked to the trade leg B and A is linked to the external confirmation C and A is linked to the index reference D
    I can only mark a reconciliation case A as ready for resolution linked to a trade leg B, an external confirmation C, and an escalation bucket D if it is not the case that A has an escalation flag and it is not the case that A is ready for resolution
    Once I mark a reconciliation case A as ready for resolution linked to a trade leg B, an external confirmation C, and an escalation bucket D, it is the case that A is ready for resolution

  mark a reconciliation case A as ready for resolution and escalate linked to a settlement event B, a counterparty reference C, and an exception category D
    I can only mark a reconciliation case A as ready for resolution and escalate linked to a settlement event B, a counterparty reference C, and an exception category D if it is the case that A has been validated and A has a verification flag and A is linked to the settlement event B and A is linked to the counterparty reference C and A is linked to the index reference D
    I can only mark a reconciliation case A as ready for resolution and escalate linked to a settlement event B, a counterparty reference C, and an exception category D if it is not the case that A is ready for resolution
    Once I mark a reconciliation case A as ready for resolution and escalate linked to a settlement event B, a counterparty reference C, and an exception category D, it is the case that A is ready for resolution and A has an escalation flag

  finalize resolution for a reconciliation case A and prepare hold linked to a trade leg B and an external confirmation C
    I can only finalize resolution for a reconciliation case A and prepare hold linked to a trade leg B and an external confirmation C if it is the case that A is ready for resolution and A has an escalation flag and A is linked to the trade leg B and A is linked to the external confirmation C

    Once I finalize resolution for a reconciliation case A and prepare hold linked to a trade leg B and an external confirmation C, it is not the case anymore that A has an escalation flag and it is not the case anymore that A has been validated and it is not the case anymore that A is under additional review flag

  reapply evidence collection for a reconciliation case A using an evidence attachment B by an operator identity C

    I can only reapply evidence collection for a reconciliation case A using an evidence attachment B by an operator identity C if it is not the case that A has been validated
    Once I reapply evidence collection for a reconciliation case A using an evidence attachment B by an operator identity C, it is the case that A has been validated

  attest attachment for a reconciliation case A with an evidence attachment B
    I can only attest attachment for a reconciliation case A with an evidence attachment B if it is the case that A is ready for resolution and A has been validated and A is linked to the evidence attachment B
    I can only attest attachment for a reconciliation case A with an evidence attachment B if it is not the case that A has an escalation flag and it is not the case that the attachment for A is attested
    Once I attest attachment for a reconciliation case A with an evidence attachment B, it is the case that the attachment for A is attested

  attest attachment for a reconciliation case A using a third party feed item B
    I can only attest attachment for a reconciliation case A using a third party feed item B if it is the case that A is ready for resolution and A has been validated and B is available in the third party feed
    I can only attest attachment for a reconciliation case A using a third party feed item B if it is not the case that A has an escalation flag and it is not the case that the attachment for A is attested
    Once I attest attachment for a reconciliation case A using a third party feed item B, it is the case that the attachment for A is attested
    Once I attest attachment for a reconciliation case A using a third party feed item B, it is not the case anymore that B is available in the third party feed

  route a reconciliation case A for approval using an approval token B
    I can only route a reconciliation case A for approval using an approval token B if it is the case that the attachment for A is attested and B is available and A is routed for approval with B
    Once I route a reconciliation case A for approval using an approval token B, it is the case that A is linked to the approval token B
    Once I route a reconciliation case A for approval using an approval token B, it is not the case anymore that B is available

  consume an approval token C for a parent case A and a related case B
    I can only consume an approval token C for a parent case A and a related case B if it is the case that A is active and A is eligible for secondary closure and A is linked to approval token C and B is linked to the approval token C
    I can only consume an approval token C for a parent case A and a related case B if it is not the case that A has consumed approval token C
    Once I consume an approval token C for a parent case A and a related case B, it is the case that A has consumed approval token C

  attest an evidence attachment B for a reconciliation case A by an operator identity C
    I can only attest an evidence attachment B for a reconciliation case A by an operator identity C if it is the case that A is active and A is eligible for secondary closure and A has been validated and the attachment for A is attested and A is linked to the evidence attachment B and C is available
    I can only attest an evidence attachment B for a reconciliation case A by an operator identity C if it is not the case that A has an operator attestation flag
    Once I attest an evidence attachment B for a reconciliation case A by an operator identity C, it is the case that A has an operator attestation flag

  close a reconciliation case A via primary route
    I can only close a reconciliation case A via primary route if it is the case that A is eligible for primary closure and A is active and A has a case assignment flag and A has a verification flag and A is ready for resolution and the attachment for A is attested and A has been validated
    I can only close a reconciliation case A via primary route if it is not the case that A is closed
    Once I close a reconciliation case A via primary route, it is the case that A is closed

  close a reconciliation case A with an approval token B
    I can only close a reconciliation case A with an approval token B if it is the case that A is eligible for secondary closure and A is active and A has a case assignment flag and A has a verification flag and A is ready for resolution and the attachment for A is attested and A has been validated and A has consumed approval token B

    I can only close a reconciliation case A with operator attestation if it is not the case that A is closed
    Once I close a reconciliation case A with operator attestation, it is the case that A is closed

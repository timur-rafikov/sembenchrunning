--- TYPE HIERARCHY ---

  Everything that is a base object is also a object
  Everything that is a reconciliation case or a payment batch or a ledger account or a counterparty account or a payment instruction or a bank statement entry or a suspense tag or a automation rule or a confirmation document or a supporting attachment or a bank reference file or a exception reason code or a operator role is also a base object
  Everything that is a resolver role or a approver role is also a operator role
  Everything that is a processing node or a source system is also a reconciliation case

--- ACTIONS ---

  open a reconciliation case A for registration
    I can only open a reconciliation case A for registration if it is not the case that the case A is registered and it is not the case that A is closed
    Once I open a reconciliation case A for registration, it is the case that the case A is registered

  assign a payment batch B to a reconciliation case A
    I can only assign a payment batch B to a reconciliation case A if it is the case that the case A is registered and B is available and A is a case batch candidate for B
    I can only assign a payment batch B to a reconciliation case A if it is not the case that the case batch is assigned to A
    Once I assign a payment batch B to a reconciliation case A, it is the case that the case A is assigned to the payment batch B and the case batch is assigned to A
    Once I assign a payment batch B to a reconciliation case A, it is not the case anymore that B is available

  unassign a payment batch B from a reconciliation case A
    I can only unassign a payment batch B from a reconciliation case A if it is the case that the case A is assigned to the payment batch B
    I can only unassign a payment batch B from a reconciliation case A if it is not the case that A is validated for the ledger and it is not the case that A resolution is in progress
    Once I unassign a payment batch B from a reconciliation case A, it is the case that B is available
    Once I unassign a payment batch B from a reconciliation case A, it is not the case anymore that the case A is assigned to the payment batch B and it is not the case anymore that the case batch is assigned to A and it is not the case anymore that A is on hold and it is not the case anymore that A has been adjudicated and it is not the case anymore that A requires approval and it is not the case anymore that A resolution is complete and it is not the case anymore that A has an accounting review flag

  apply a suspense tag B to a reconciliation case A
    I can only apply a suspense tag B to a reconciliation case A if it is the case that the case A is registered and suspense tag available B
    Once I apply a suspense tag B to a reconciliation case A, it is the case that A has a suspense tag B
    Once I apply a suspense tag B to a reconciliation case A, it is not the case anymore that suspense tag available B

  remove a suspense tag B from a reconciliation case A
    I can only remove a suspense tag B from a reconciliation case A if it is the case that A has a suspense tag B
    Once I remove a suspense tag B from a reconciliation case A, it is the case that suspense tag available B
    Once I remove a suspense tag B from a reconciliation case A, it is not the case anymore that A has a suspense tag B

  apply hold from a suspense tag B to a reconciliation case A
    I can only apply hold from a suspense tag B to a reconciliation case A if it is the case that the case A is registered and the case batch is assigned to A and A has a suspense tag B
    I can only apply hold from a suspense tag B to a reconciliation case A if it is not the case that A is on hold
    Once I apply hold from a suspense tag B to a reconciliation case A, it is the case that A is on hold

  apply automated hold and flag approval for a reconciliation case A using an automation rule B
    I can only apply automated hold and flag approval for a reconciliation case A using an automation rule B if it is the case that the case A is registered and the case batch is assigned to A and B is an automation rule available
    I can only apply automated hold and flag approval for a reconciliation case A using an automation rule B if it is not the case that A is on hold
    Once I apply automated hold and flag approval for a reconciliation case A using an automation rule B, it is the case that A is on hold and A requires approval
    Once I apply automated hold and flag approval for a reconciliation case A using an automation rule B, it is not the case anymore that B is an automation rule available

  adjudicate a reconciliation case A with a suspense tag B and a supporting attachment C
    I can only adjudicate a reconciliation case A with a suspense tag B and a supporting attachment C if it is the case that A is on hold and the case batch is assigned to A and A has a suspense tag B and C is a supporting attachment available
    I can only adjudicate a reconciliation case A with a suspense tag B and a supporting attachment C if it is not the case that A has been adjudicated
    Once I adjudicate a reconciliation case A with a suspense tag B and a supporting attachment C, it is the case that A has been adjudicated
    Once I adjudicate a reconciliation case A with a suspense tag B and a supporting attachment C, it is not the case anymore that A requires approval

  adjudicate a reconciliation case A with a confirmation document B
    I can only adjudicate a reconciliation case A with a confirmation document B if it is the case that the case batch is assigned to A and case confirmation for A is recorded from the source with B
    I can only adjudicate a reconciliation case A with a confirmation document B if it is not the case that A has been adjudicated
    Once I adjudicate a reconciliation case A with a confirmation document B, it is the case that A has been adjudicated
    Once I adjudicate a reconciliation case A with a confirmation document B, it is not the case anymore that A requires approval

  link a payment instruction B to a reconciliation case A

    Once I link a payment instruction B to a reconciliation case A, it is the case that A has a payment instruction B
    Once I link a payment instruction B to a reconciliation case A, it is not the case anymore that B is available

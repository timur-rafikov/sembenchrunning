DOMAIN IN NATURAL LANGUAGE

--- TYPE HIERARCHY ---

  Everything that is a transaction or a submitter or a primary approver group or a compliance reviewer or a business reviewer or a operations reviewer or a sanctions service or a delegated approver or a approval step or a sanctions analyst or a investigative document or a counterparty profile or a escalation target is also a object
  Everything that is a senior approver or a executive approver is also a escalation target
  Everything that is a domestic transaction or a cross border transaction is also a transaction

--- ACTIONS ---

  open a transaction A for intake
    I can only open a transaction A for intake if it is not the case that A is opened and it is not the case that A is authorized
    Once I open a transaction A for intake, it is the case that A is opened

  submit a transaction A by a submitter B
    I can only submit a transaction A by a submitter B if it is the case that A is opened and B is available and B is eligible to submit A
    I can only submit a transaction A by a submitter B if it is not the case that the submitter is linked to A
    Once I submit a transaction A by a submitter B, it is the case that A is submitted by B and the submitter is linked to A
    Once I submit a transaction A by a submitter B, it is not the case anymore that B is available

  withdraw submission of a transaction A by a submitter B
    I can only withdraw submission of a transaction A by a submitter B if it is the case that A is submitted by B
    I can only withdraw submission of a transaction A by a submitter B if it is not the case that primary approval for A is recorded and it is not the case that dual approval for A is requested
    Once I withdraw submission of a transaction A by a submitter B, it is the case that B is available
    Once I withdraw submission of a transaction A by a submitter B, it is not the case anymore that A is submitted by B and it is not the case anymore that the submitter is linked to A and it is not the case anymore that screening is assigned to A and it is not the case anymore that screening is cleared for A and it is not the case anymore that A requires manual analyst review and it is not the case anymore that dual approval is recorded for A and it is not the case anymore that escalation is requested for A

  assign a sanctions service B to a transaction A
    I can only assign a sanctions service B to a transaction A if it is the case that A is opened and B is available as the sanctions service
    Once I assign a sanctions service B to a transaction A, it is the case that A is assigned to the sanctions service B
    Once I assign a sanctions service B to a transaction A, it is not the case anymore that B is available as the sanctions service

  unassign a sanctions service B from a transaction A
    I can only unassign a sanctions service B from a transaction A if it is the case that A is assigned to the sanctions service B
    Once I unassign a sanctions service B from a transaction A, it is the case that B is available as the sanctions service
    Once I unassign a sanctions service B from a transaction A, it is not the case anymore that A is assigned to the sanctions service B

  record automated screening result for a transaction A using a sanctions service B
    I can only record automated screening result for a transaction A using a sanctions service B if it is the case that A is opened and the submitter is linked to A and A is assigned to the sanctions service B
    I can only record automated screening result for a transaction A using a sanctions service B if it is not the case that screening is assigned to A
    Once I record automated screening result for a transaction A using a sanctions service B, it is the case that screening is assigned to A

  assign a delegated approver B for screening of a transaction A
    I can only assign a delegated approver B for screening of a transaction A if it is the case that A is opened and the submitter is linked to A and B is available as the delegated approver
    I can only assign a delegated approver B for screening of a transaction A if it is not the case that screening is assigned to A
    Once I assign a delegated approver B for screening of a transaction A, it is the case that screening is assigned to A and A requires manual analyst review
    Once I assign a delegated approver B for screening of a transaction A, it is not the case anymore that B is available as the delegated approver

  validate screening for a transaction A using a sanctions service B by a sanctions analyst C
    I can only validate screening for a transaction A using a sanctions service B by a sanctions analyst C if it is the case that screening is assigned to A and the submitter is linked to A and A is assigned to the sanctions service B and C is available as a sanctions analyst
    I can only validate screening for a transaction A using a sanctions service B by a sanctions analyst C if it is not the case that screening is cleared for A
    Once I validate screening for a transaction A using a sanctions service B by a sanctions analyst C, it is the case that screening is cleared for A
    Once I validate screening for a transaction A using a sanctions service B by a sanctions analyst C, it is not the case anymore that A requires manual analyst review

  approve screening clearance for a transaction A at an approval step B
    I can only approve screening clearance for a transaction A at an approval step B if it is the case that the submitter is linked to A and cross border interlock is linked to A at B
    I can only approve screening clearance for a transaction A at an approval step B if it is not the case that screening is cleared for A
    Once I approve screening clearance for a transaction A at an approval step B, it is the case that screening is cleared for A
    Once I approve screening clearance for a transaction A at an approval step B, it is not the case anymore that A requires manual analyst review

  assign a business reviewer B to review a transaction A
    I can only assign a business reviewer B to review a transaction A if it is the case that A is opened and B is available as the business reviewer and B is eligible as the business reviewer for A
    Once I assign a business reviewer B to review a transaction A, it is the case that B is assigned to review A
    Once I assign a business reviewer B to review a transaction A, it is not the case anymore that B is available as the business reviewer

  unassign a business reviewer B from reviewing a transaction A
    I can only unassign a business reviewer B from reviewing a transaction A if it is the case that B is assigned to review A
    Once I unassign a business reviewer B from reviewing a transaction A, it is the case that B is available as the business reviewer
    Once I unassign a business reviewer B from reviewing a transaction A, it is not the case anymore that B is assigned to review A

  assign a compliance reviewer B to review compliance for a transaction A
    I can only assign a compliance reviewer B to review compliance for a transaction A if it is the case that A is opened and B is available as the compliance reviewer and B is eligible as the compliance reviewer for A
    Once I assign a compliance reviewer B to review compliance for a transaction A, it is the case that B is assigned to review compliance for A
    Once I assign a compliance reviewer B to review compliance for a transaction A, it is not the case anymore that B is available as the compliance reviewer

  unassign a compliance reviewer B from reviewing compliance for a transaction A
    I can only unassign a compliance reviewer B from reviewing compliance for a transaction A if it is the case that B is assigned to review compliance for A
    Once I unassign a compliance reviewer B from reviewing compliance for a transaction A, it is the case that B is available as the compliance reviewer
    Once I unassign a compliance reviewer B from reviewing compliance for a transaction A, it is not the case anymore that B is assigned to review compliance for A

  assign an operations reviewer B to review operations for a transaction A

    Once I assign an operations reviewer B to review operations for a transaction A, it is the case that B is assigned to review operations for A
    Once I assign an operations reviewer B to review operations for a transaction A, it is not the case anymore that B is available as the operations reviewer

  unassign an operations reviewer B from reviewing operations for a transaction A
    I can only unassign an operations reviewer B from reviewing operations for a transaction A if it is the case that B is assigned to review operations for A
    Once I unassign an operations reviewer B from reviewing operations for a transaction A, it is the case that B is available as the operations reviewer
    Once I unassign an operations reviewer B from reviewing operations for a transaction A, it is not the case anymore that B is assigned to review operations for A

  assign a counterparty profile B to a transaction A
    I can only assign a counterparty profile B to a transaction A if it is the case that A is opened and B is available as a counterparty profile and A has the counterparty profile B
    Once I assign a counterparty profile B to a transaction A, it is the case that B is assigned to the counterparty profile for A
    Once I assign a counterparty profile B to a transaction A, it is not the case anymore that B is available as a counterparty profile

  unassign a counterparty profile B from a transaction A
    I can only unassign a counterparty profile B from a transaction A if it is the case that B is assigned to the counterparty profile for A
    Once I unassign a counterparty profile B from a transaction A, it is the case that B is available as a counterparty profile
    Once I unassign a counterparty profile B from a transaction A, it is not the case anymore that B is assigned to the counterparty profile for A

  record primary approval for a transaction A by a primary approver group B at an approval step C with a business reviewer D
    I can only record primary approval for a transaction A by a primary approver group B at an approval step C with a business reviewer D if it is the case that the submitter is linked to A and C is assigned to review A and D is assigned to review compliance for A and B is available as the primary approver and B is eligible as the primary approver for A
    I can only record primary approval for a transaction A by a primary approver group B at an approval step C with a business reviewer D if it is not the case that primary approval for A is recorded
    Once I record primary approval for a transaction A by a primary approver group B at an approval step C with a business reviewer D, it is the case that A has primary approval by the approver group at B and primary approval for A is recorded
    Once I record primary approval for a transaction A by a primary approver group B at an approval step C with a business reviewer D, it is not the case anymore that B is available as the primary approver

  record primary approval with investigation for a transaction A by a primary approver group B with an operations reviewer C using an investigative document D
    I can only record primary approval with investigation for a transaction A by a primary approver group B with an operations reviewer C using an investigative document D if it is the case that the submitter is linked to A and C is assigned to review operations for A and D is available as an investigative document and B is available as the primary approver and B is eligible as the primary approver for A and A has the investigative document D
    I can only record primary approval with investigation for a transaction A by a primary approver group B with an operations reviewer C using an investigative document D if it is not the case that primary approval for A is recorded
    Once I record primary approval with investigation for a transaction A by a primary approver group B with an operations reviewer C using an investigative document D, it is the case that A has primary approval by the approver group at B and primary approval for A is recorded and escalation is requested for A and A requires manual analyst review
    Once I record primary approval with investigation for a transaction A by a primary approver group B with an operations reviewer C using an investigative document D, it is not the case anymore that B is available as the primary approver and it is not the case anymore that D is available as an investigative document

  acknowledge escalation marker for a transaction A with a business reviewer B and a compliance reviewer C
    I can only acknowledge escalation marker for a transaction A with a business reviewer B and a compliance reviewer C if it is the case that primary approval for A is recorded and escalation is requested for A and B is assigned to review A and C is assigned to review compliance for A
    Once I acknowledge escalation marker for a transaction A with a business reviewer B and a compliance reviewer C, it is not the case anymore that escalation is requested for A and it is not the case anymore that A requires manual analyst review

  request dual approval for a transaction A with a business reviewer B, a compliance reviewer C, and a senior approver D
    I can only request dual approval for a transaction A with a business reviewer B, a compliance reviewer C, and a senior approver D if it is the case that screening is cleared for A and primary approval for A is recorded and B is assigned to review A and C is assigned to review compliance for A and D is the escalation target for A
    I can only request dual approval for a transaction A with a business reviewer B, a compliance reviewer C, and a senior approver D if it is not the case that A requires manual analyst review and it is not the case that dual approval for A is requested
    Once I request dual approval for a transaction A with a business reviewer B, a compliance reviewer C, and a senior approver D, it is the case that dual approval for A is requested

  request dual approval with escalation for a transaction A involving an operations reviewer B, a counterparty profile C, and an executive approver D
    I can only request dual approval with escalation for a transaction A involving an operations reviewer B, a counterparty profile C, and an executive approver D if it is the case that screening is cleared for A and primary approval for A is recorded and B is assigned to review operations for A and C is assigned to the counterparty profile for A and D is the escalation target for A
    I can only request dual approval with escalation for a transaction A involving an operations reviewer B, a counterparty profile C, and an executive approver D if it is not the case that dual approval for A is requested
    Once I request dual approval with escalation for a transaction A involving an operations reviewer B, a counterparty profile C, and an executive approver D, it is the case that dual approval for A is requested and A requires manual analyst review

  complete dual approval for a transaction A with a business reviewer B and a compliance reviewer C
    I can only complete dual approval for a transaction A with a business reviewer B and a compliance reviewer C if it is the case that dual approval for A is requested and A requires manual analyst review and B is assigned to review A and C is assigned to review compliance for A
    Once I complete dual approval for a transaction A with a business reviewer B and a compliance reviewer C, it is the case that dual approval is recorded for A
    Once I complete dual approval for a transaction A with a business reviewer B and a compliance reviewer C, it is not the case anymore that A requires manual analyst review and it is not the case anymore that screening is cleared for A and it is not the case anymore that escalation is requested for A

  restore screening clearance for a transaction A after escalation with a sanctions service B and a sanctions analyst C
    I can only restore screening clearance for a transaction A after escalation with a sanctions service B and a sanctions analyst C if it is the case that dual approval is recorded for A and dual approval for A is requested and the submitter is linked to A and A is assigned to the sanctions service B and C is available as a sanctions analyst
    I can only restore screening clearance for a transaction A after escalation with a sanctions service B and a sanctions analyst C if it is not the case that screening is cleared for A
    Once I restore screening clearance for a transaction A after escalation with a sanctions service B and a sanctions analyst C, it is the case that screening is cleared for A

  mark secondary approver as assigned for a transaction A with a sanctions service B

    I can only mark secondary approver as assigned for a transaction A with a sanctions service B if it is not the case that A requires manual analyst review and it is not the case that the secondary approver for A is confirmed
    Once I mark secondary approver as assigned for a transaction A with a sanctions service B, it is the case that the secondary approver for A is confirmed

  mark a delegated approver B as secondary approver for a transaction A
    I can only mark a delegated approver B as secondary approver for a transaction A if it is the case that dual approval for A is requested and screening is cleared for A and B is available as the delegated approver
    I can only mark a delegated approver B as secondary approver for a transaction A if it is not the case that A requires manual analyst review and it is not the case that the secondary approver for A is confirmed
    Once I mark a delegated approver B as secondary approver for a transaction A, it is the case that the secondary approver for A is confirmed
    Once I mark a delegated approver B as secondary approver for a transaction A, it is not the case anymore that B is available as the delegated approver

  bind a transaction A to an approval step B for secondary approval
    I can only bind a transaction A to an approval step B for secondary approval if it is the case that the secondary approver for A is confirmed and B is available for approval and A is bound to the transaction step B
    Once I bind a transaction A to an approval step B for secondary approval, it is the case that A requires secondary approval by B
    Once I bind a transaction A to an approval step B for secondary approval, it is not the case anymore that B is available for approval

  link cross border interlock for a cross border transaction A with a domestic transaction B at an approval step C
    I can only link cross border interlock for a cross border transaction A with a domestic transaction B at an approval step C if it is the case that A is opened and A is a cross border transaction and cross border interlock is required for A at C and B requires secondary approval by C
    I can only link cross border interlock for a cross border transaction A with a domestic transaction B at an approval step C if it is not the case that cross border interlock is linked to A at C
    Once I link cross border interlock for a cross border transaction A with a domestic transaction B at an approval step C, it is the case that cross border interlock is linked to A at C

  mark final checks ready for a transaction A using a sanctions service B with a sanctions analyst C
    I can only mark final checks ready for a transaction A using a sanctions service B with a sanctions analyst C if it is the case that A is opened and A is a cross border transaction and screening is cleared for A and the secondary approver for A is confirmed and A is assigned to the sanctions service B and C is available as a sanctions analyst
    I can only mark final checks ready for a transaction A using a sanctions service B with a sanctions analyst C if it is not the case that final checks are signed off for A
    Once I mark final checks ready for a transaction A using a sanctions service B with a sanctions analyst C, it is the case that final checks are signed off for A

  finalize domestic authorization for a transaction A
    I can only finalize domestic authorization for a transaction A if it is the case that A is a domestic transaction and A is opened and the submitter is linked to A and primary approval for A is recorded and dual approval for A is requested and the secondary approver for A is confirmed and screening is cleared for A
    I can only finalize domestic authorization for a transaction A if it is not the case that A is authorized
    Once I finalize domestic authorization for a transaction A, it is the case that A is authorized

  finalize authorization for a transaction A with an approval step B
    I can only finalize authorization for a transaction A with an approval step B if it is the case that A is a cross border transaction and A is opened and the submitter is linked to A and primary approval for A is recorded and dual approval for A is requested and the secondary approver for A is confirmed and screening is cleared for A and cross border interlock is linked to A at B
    I can only finalize authorization for a transaction A with an approval step B if it is not the case that A is authorized
    Once I finalize authorization for a transaction A with an approval step B, it is the case that A is authorized

  finalize authorization for a transaction A with final checks

    I can only finalize authorization for a transaction A with final checks if it is not the case that A is authorized

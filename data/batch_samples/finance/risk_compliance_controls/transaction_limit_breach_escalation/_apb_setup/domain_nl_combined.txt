DOMAIN IN NATURAL LANGUAGE
============================================================

--- TYPE HIERARCHY ---

  Everything that is a transaction case or a originating entity or a transaction profile or a limit rule or a customer profile or a product profile or a screening alert or a automated detector or a approver role or a specialist or a evidence document or a risk indicator or a auxiliary object is also a object
  Everything that is a escalation channel or a escalation channel secondary is also a auxiliary object
  Everything that is a case subtype primary or a case subtype secondary is also a transaction case

--- ACTIONS ---

  create a transaction case A
    I can only create a transaction case A if it is not the case that A is open and it is not the case that A is closed
    Once I create a transaction case A, it is the case that A is open

  associate a transaction case A with an originating entity B
    I can only associate a transaction case A with an originating entity B if it is the case that A is open and the origin is available for B and A is compatible with the originating entity B
    I can only associate a transaction case A with an originating entity B if it is not the case that A has an active association
    Once I associate a transaction case A with an originating entity B, it is the case that A is linked to the originating entity B and A has an active association
    Once I associate a transaction case A with an originating entity B, it is not the case anymore that the origin is available for B

  reset associations for a transaction case A with an originating entity B
    I can only reset associations for a transaction case A with an originating entity B if it is the case that A is linked to the originating entity B
    I can only reset associations for a transaction case A with an originating entity B if it is not the case that the decision for A has been recorded and it is not the case that the approval for A has been assigned
    Once I reset associations for a transaction case A with an originating entity B, it is the case that the origin is available for B
    Once I reset associations for a transaction case A with an originating entity B, it is not the case anymore that A is linked to the originating entity B and it is not the case anymore that A has an active association and it is not the case anymore that A is on hold and it is not the case anymore that A has completed specialist review and it is not the case anymore that a specialist is required for A and it is not the case anymore that approval has been granted for A and it is not the case anymore that an additional check is required for A

  reserve a screening alert B for a transaction case A
    I can only reserve a screening alert B for a transaction case A if it is the case that A is open and the screening alert is available for B
    Once I reserve a screening alert B for a transaction case A, it is the case that A has a reserved alert for B
    Once I reserve a screening alert B for a transaction case A, it is not the case anymore that the screening alert is available for B

  release a screening alert B from a transaction case A
    I can only release a screening alert B from a transaction case A if it is the case that A has a reserved alert for B
    Once I release a screening alert B from a transaction case A, it is the case that the screening alert is available for B
    Once I release a screening alert B from a transaction case A, it is not the case anymore that A has a reserved alert for B

  apply alert hold to a transaction case A for a screening alert B
    I can only apply alert hold to a transaction case A for a screening alert B if it is the case that A is open and A has an active association and A has a reserved alert for B
    I can only apply alert hold to a transaction case A for a screening alert B if it is not the case that A is on hold
    Once I apply alert hold to a transaction case A for a screening alert B, it is the case that A is on hold

  apply detector hold to a transaction case A using an automated detector B
    I can only apply detector hold to a transaction case A using an automated detector B if it is the case that A is open and A has an active association and the detector is available for B
    I can only apply detector hold to a transaction case A using an automated detector B if it is not the case that A is on hold
    Once I apply detector hold to a transaction case A using an automated detector B, it is the case that A is on hold and a specialist is required for A
    Once I apply detector hold to a transaction case A using an automated detector B, it is not the case anymore that the detector is available for B

  specialist a specialist C captures evidence for a transaction case A with a screening alert B
    I can only specialist a specialist C captures evidence for a transaction case A with a screening alert B if it is the case that A is on hold and A has an active association and A has a reserved alert for B and C is available
    I can only specialist a specialist C captures evidence for a transaction case A with a screening alert B if it is not the case that A has completed specialist review
    Once I specialist a specialist C captures evidence for a transaction case A with a screening alert B, it is the case that A has completed specialist review
    Once I specialist a specialist C captures evidence for a transaction case A with a screening alert B, it is not the case anymore that a specialist is required for A

  specialist captures evidence for a transaction case A via an approver role B
    I can only specialist captures evidence for a transaction case A via an approver role B if it is the case that A has an active association and A is confirmed by the approver role B
    I can only specialist captures evidence for a transaction case A via an approver role B if it is not the case that A has completed specialist review
    Once I specialist captures evidence for a transaction case A via an approver role B, it is the case that A has completed specialist review
    Once I specialist captures evidence for a transaction case A via an approver role B, it is not the case anymore that a specialist is required for A

  bind a transaction case A to a customer profile B
    I can only bind a transaction case A to a customer profile B if it is the case that A is open and the customer profile is available for B and A is compatible with the customer profile B
    Once I bind a transaction case A to a customer profile B, it is the case that A is bound to the customer profile B
    Once I bind a transaction case A to a customer profile B, it is not the case anymore that the customer profile is available for B

  release a customer profile B from a transaction case A
    I can only release a customer profile B from a transaction case A if it is the case that A is bound to the customer profile B
    Once I release a customer profile B from a transaction case A, it is the case that the customer profile is available for B
    Once I release a customer profile B from a transaction case A, it is not the case anymore that A is bound to the customer profile B

  bind a limit rule B to a transaction case A
    I can only bind a limit rule B to a transaction case A if it is the case that A is open and the limit rule is available for B and A is compatible with the limit rule B
    Once I bind a limit rule B to a transaction case A, it is the case that A is bound to the limit rule B
    Once I bind a limit rule B to a transaction case A, it is not the case anymore that the limit rule is available for B

  release a limit rule B from a transaction case A
    I can only release a limit rule B from a transaction case A if it is the case that A is bound to the limit rule B
    Once I release a limit rule B from a transaction case A, it is the case that the limit rule is available for B
    Once I release a limit rule B from a transaction case A, it is not the case anymore that A is bound to the limit rule B

  bind a product profile B to a transaction case A

    Once I bind a product profile B to a transaction case A, it is the case that A is bound to the product profile B
    Once I bind a product profile B to a transaction case A, it is not the case anymore that the product profile is available for B

  release a product profile B from a transaction case A
    I can only release a product profile B from a transaction case A if it is the case that A is bound to the product profile B
    Once I release a product profile B from a transaction case A, it is the case that the product profile is available for B
    Once I release a product profile B from a transaction case A, it is not the case anymore that A is bound to the product profile B

  bind a risk indicator B to a transaction case A
    I can only bind a risk indicator B to a transaction case A if it is the case that A is open and B is available and A is compatible with the risk indicator B
    Once I bind a risk indicator B to a transaction case A, it is the case that A is bound to the risk indicator B
    Once I bind a risk indicator B to a transaction case A, it is not the case anymore that B is available

  release a risk indicator B from a transaction case A
    I can only release a risk indicator B from a transaction case A if it is the case that A is bound to the risk indicator B
    Once I release a risk indicator B from a transaction case A, it is the case that B is available
    Once I release a risk indicator B from a transaction case A, it is not the case anymore that A is bound to the risk indicator B

  perform risk assessment on a transaction case A using a transaction profile B, a customer profile C, and a limit rule D
    I can only perform risk assessment on a transaction case A using a transaction profile B, a customer profile C, and a limit rule D if it is the case that A has an active association and A is bound to the customer profile C and A is bound to the limit rule D and the profile is available for B and A is compatible with the transaction profile B
    I can only perform risk assessment on a transaction case A using a transaction profile B, a customer profile C, and a limit rule D if it is not the case that the decision for A has been recorded
    Once I perform risk assessment on a transaction case A using a transaction profile B, a customer profile C, and a limit rule D, it is the case that A has a decision flag for the transaction profile B and the decision for A has been recorded
    Once I perform risk assessment on a transaction case A using a transaction profile B, a customer profile C, and a limit rule D, it is not the case anymore that the profile is available for B

  perform extended risk assessment on a transaction case A using a transaction profile B, a product profile C, and an evidence document D
    I can only perform extended risk assessment on a transaction case A using a transaction profile B, a product profile C, and an evidence document D if it is the case that A has an active association and A is bound to the product profile C and D is available and the profile is available for B and A is compatible with the transaction profile B and A is compatible with the evidence document D
    I can only perform extended risk assessment on a transaction case A using a transaction profile B, a product profile C, and an evidence document D if it is not the case that the decision for A has been recorded
    Once I perform extended risk assessment on a transaction case A using a transaction profile B, a product profile C, and an evidence document D, it is the case that A has a decision flag for the transaction profile B and the decision for A has been recorded and an additional check is required for A and a specialist is required for A
    Once I perform extended risk assessment on a transaction case A using a transaction profile B, a product profile C, and an evidence document D, it is not the case anymore that the profile is available for B and it is not the case anymore that D is available

  clear additional checks for a transaction case A using a customer profile B and a limit rule C
    I can only clear additional checks for a transaction case A using a customer profile B and a limit rule C if it is the case that the decision for A has been recorded and an additional check is required for A and A is bound to the customer profile B and A is bound to the limit rule C
    Once I clear additional checks for a transaction case A using a customer profile B and a limit rule C, it is not the case anymore that an additional check is required for A and it is not the case anymore that a specialist is required for A

  allocate approver for a transaction case A using a customer profile B, a limit rule C, and an escalation channel D
    I can only allocate approver for a transaction case A using a customer profile B, a limit rule C, and an escalation channel D if it is the case that A has completed specialist review and the decision for A has been recorded and A is bound to the customer profile B and A is bound to the limit rule C and A is bound to the auxiliary object D
    I can only allocate approver for a transaction case A using a customer profile B, a limit rule C, and an escalation channel D if it is not the case that a specialist is required for A and it is not the case that the approval for A has been assigned
    Once I allocate approver for a transaction case A using a customer profile B, a limit rule C, and an escalation channel D, it is the case that the approval for A has been assigned

  allocate escalation approver for a transaction case A using a product profile B, a risk indicator C, and an escalation channel secondary D
    I can only allocate escalation approver for a transaction case A using a product profile B, a risk indicator C, and an escalation channel secondary D if it is the case that A has completed specialist review and the decision for A has been recorded and A is bound to the product profile B and A is bound to the risk indicator C and A is bound to the auxiliary object D
    I can only allocate escalation approver for a transaction case A using a product profile B, a risk indicator C, and an escalation channel secondary D if it is not the case that the approval for A has been assigned
    Once I allocate escalation approver for a transaction case A using a product profile B, a risk indicator C, and an escalation channel secondary D, it is the case that the approval for A has been assigned and a specialist is required for A

  finalize approval stage for a transaction case A with a customer profile B and a limit rule C

    Once I finalize approval stage for a transaction case A with a customer profile B and a limit rule C, it is not the case anymore that a specialist is required for A and it is not the case anymore that A has completed specialist review and it is not the case anymore that an additional check is required for A

  reassign specialist review for a transaction case A with a screening alert B to a specialist C

    I can only reassign specialist review for a transaction case A with a screening alert B to a specialist C if it is not the case that A has completed specialist review
    Once I reassign specialist review for a transaction case A with a screening alert B to a specialist C, it is the case that A has completed specialist review

  mark a transaction case A with a screening alert B as escalation ready

    I can only mark a transaction case A with a screening alert B as escalation ready if it is not the case that a specialist is required for A and it is not the case that the approver is ready for A

  mark a transaction case A as escalation ready using an automated detector B

    I can only mark a transaction case A as escalation ready using an automated detector B if it is not the case that a specialist is required for A and it is not the case that the approver is ready for A
    Once I mark a transaction case A as escalation ready using an automated detector B, it is the case that the approver is ready for A
    Once I mark a transaction case A as escalation ready using an automated detector B, it is not the case anymore that the detector is available for B

  assign an approver role B to a transaction case A
    I can only assign an approver role B to a transaction case A if it is the case that the approver is ready for A and the approver role is available for B and the primary approver role for A is compatible with B
    Once I assign an approver role B to a transaction case A, it is the case that B is assigned to the transaction case A
    Once I assign an approver role B to a transaction case A, it is not the case anymore that the approver role is available for B

  confirm assignment of an approver role C to a case subtype secondary A for a case subtype primary B
    I can only confirm assignment of an approver role C to a case subtype secondary A for a case subtype primary B if it is the case that A is open and A is eligible for secondary closure and A has secondary compatibility with the approver role C and C is assigned to the transaction case B
    I can only confirm assignment of an approver role C to a case subtype secondary A for a case subtype primary B if it is not the case that A is confirmed by the approver role C
    Once I confirm assignment of an approver role C to a case subtype secondary A for a case subtype primary B, it is the case that A is confirmed by the approver role C

  create escalation ticket for a transaction case A with a screening alert B by a specialist C
    I can only create escalation ticket for a transaction case A with a screening alert B by a specialist C if it is the case that A is open and A is eligible for secondary closure and A has completed specialist review and the approver is ready for A and A has a reserved alert for B and C is available
    I can only create escalation ticket for a transaction case A with a screening alert B by a specialist C if it is not the case that an escalation ticket is created for A
    Once I create escalation ticket for a transaction case A with a screening alert B by a specialist C, it is the case that an escalation ticket is created for A

  close case primary for a transaction case A
    I can only close case primary for a transaction case A if it is the case that A is eligible for primary closure and A is open and A has an active association and the decision for A has been recorded and the approval for A has been assigned and the approver is ready for A and A has completed specialist review
    I can only close case primary for a transaction case A if it is not the case that A is closed
    Once I close case primary for a transaction case A, it is the case that A is closed

  close case with approver for a transaction case A confirmed by an approver role B
    I can only close case with approver for a transaction case A confirmed by an approver role B if it is the case that A is eligible for secondary closure and A is open and A has an active association and the decision for A has been recorded and the approval for A has been assigned and the approver is ready for A and A has completed specialist review and A is confirmed by the approver role B
    I can only close case with approver for a transaction case A confirmed by an approver role B if it is not the case that A is closed

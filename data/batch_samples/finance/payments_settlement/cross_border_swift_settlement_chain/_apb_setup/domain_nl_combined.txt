DOMAIN IN NATURAL LANGUAGE

--- TYPE HIERARCHY ---

  Everything that is a domain object is also a object
  Everything that is a settlement case or a originating ledger or a settlement rail or a beneficiary correspondent or a currency or a nostro account or a communication channel or a approval token or a correspondent endpoint or a validation service or a liquidity reserve slot or a currency route or a routing profile is also a domain object
  Everything that is a routing profile domestic or a routing profile international is also a routing profile
  Everything that is a outgoing case or a incoming case is also a settlement case

--- ACTIONS ---

  create a settlement case A as a registered case
    I can only create a settlement case A as a registered case if it is not the case that A is a registered case and it is not the case that A is finalized
    Once I create a settlement case A as a registered case, it is the case that A is a registered case

  allocate an originating ledger B to a settlement case A
    I can only allocate an originating ledger B to a settlement case A if it is the case that A is a registered case and the originating ledger B is available and the settlement case A is allowed for the originating ledger B
    I can only allocate an originating ledger B to a settlement case A if it is not the case that the originating ledger for A is allocated
    Once I allocate an originating ledger B to a settlement case A, it is the case that A is allocated to the originating ledger B and the originating ledger for A is allocated
    Once I allocate an originating ledger B to a settlement case A, it is not the case anymore that the originating ledger B is available

  release an originating ledger B from a settlement case A
    I can only release an originating ledger B from a settlement case A if it is the case that A is allocated to the originating ledger B
    I can only release an originating ledger B from a settlement case A if it is not the case that the execution for A is scheduled and it is not the case that the approval for A is recorded
    Once I release an originating ledger B from a settlement case A, it is the case that the originating ledger B is available
    Once I release an originating ledger B from a settlement case A, it is not the case anymore that A is allocated to the originating ledger B and it is not the case anymore that the originating ledger for A is allocated and it is not the case anymore that the communication channel for A is ready and it is not the case anymore that A has been validated and it is not the case anymore that A has an approval flag and it is not the case anymore that A is authorized and it is not the case anymore that A has a post execution flag

  bind a communication channel B to a settlement case A
    I can only bind a communication channel B to a settlement case A if it is the case that A is a registered case and the communication channel B is available
    Once I bind a communication channel B to a settlement case A, it is the case that A is bound to the communication channel B
    Once I bind a communication channel B to a settlement case A, it is not the case anymore that the communication channel B is available

  unbind a communication channel B from a settlement case A

    Once I unbind a communication channel B from a settlement case A, it is the case that the communication channel B is available

  confirm the binding of a communication channel B for a settlement case A

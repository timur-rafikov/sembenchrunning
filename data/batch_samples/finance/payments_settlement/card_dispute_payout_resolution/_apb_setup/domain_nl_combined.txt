  Everything that is a dispute case internal or a dispute case external is also a dispute case

--- ACTIONS ---

  register a dispute case A as a registered case
    I can only register a dispute case A as a registered case if it is not the case that A is a registered case and it is not the case that A is finalized
    Once I register a dispute case A as a registered case, it is the case that A is a registered case

  assign a dispute case A to a payout rail B
    I can only assign a dispute case A to a payout rail B if it is the case that A is a registered case and B payout rail is available and A case is supported by B rail
    I can only assign a dispute case A to a payout rail B if it is not the case that the rail allocation for A is committed
    Once I assign a dispute case A to a payout rail B, it is the case that A is assigned to the payout rail B and the rail allocation for A is committed
    Once I assign a dispute case A to a payout rail B, it is not the case anymore that B payout rail is available

  release a dispute case A from a payout rail B
    I can only release a dispute case A from a payout rail B if it is the case that A is assigned to the payout rail B
    I can only release a dispute case A from a payout rail B if it is not the case that an execution plan for A is committed and it is not the case that authorization for A is verified
    Once I release a dispute case A from a payout rail B, it is the case that B payout rail is available
    Once I release a dispute case A from a payout rail B, it is not the case anymore that A is assigned to the payout rail B and it is not the case anymore that the rail allocation for A is committed and it is not the case anymore that the refund account for A is confirmed and it is not the case anymore that compliance for A is verified and it is not the case anymore that approval token is present for A and it is not the case anymore that authorization is finalized for A and it is not the case anymore that the clearing required flag for A

  reserve an external refund account B for a dispute case A
    I can only reserve an external refund account B for a dispute case A if it is the case that A is a registered case and B external refund account is available
    Once I reserve an external refund account B for a dispute case A, it is the case that A is bound to the external refund account B
    Once I reserve an external refund account B for a dispute case A, it is not the case anymore that B external refund account is available

  release an external refund account B from a dispute case A
    I can only release an external refund account B from a dispute case A if it is the case that A is bound to the external refund account B
    Once I release an external refund account B from a dispute case A, it is the case that B external refund account is available
    Once I release an external refund account B from a dispute case A, it is not the case anymore that A is bound to the external refund account B

  confirm the binding of an external refund account B for a dispute case A
    I can only confirm the binding of an external refund account B for a dispute case A if it is the case that A is a registered case and the rail allocation for A is committed and A is bound to the external refund account B
    I can only confirm the binding of an external refund account B for a dispute case A if it is not the case that the refund account for A is confirmed

  consume a manual approval token B for a dispute case A

    Once I consume a manual approval token B for a dispute case A, it is not the case anymore that B manual approval token is available

  apply a compliance record C to a dispute case A with an external refund account B
    I can only apply a compliance record C to a dispute case A with an external refund account B if it is the case that the refund account for A is confirmed and the rail allocation for A is committed and A is bound to the external refund account B and C compliance record is available
    I can only apply a compliance record C to a dispute case A with an external refund account B if it is not the case that compliance for A is verified
    Once I apply a compliance record C to a dispute case A with an external refund account B, it is the case that compliance for A is verified
    Once I apply a compliance record C to a dispute case A with an external refund account B, it is not the case anymore that approval token is present for A

  verify compliance for a dispute case A using a payout instruction template B

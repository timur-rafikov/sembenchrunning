--- ACTIONS ---

  initialize a netting batch A
    I can only initialize a netting batch A if it is not the case that the batch A is initialized and it is not the case that A is released
    Once I initialize a netting batch A, it is the case that the batch A is initialized

  attach a payment instruction B to a netting batch A
    I can only attach a payment instruction B to a netting batch A if it is the case that the batch A is initialized and instruction is available for B and B is eligible for netting batch A
    I can only attach a payment instruction B to a netting batch A if it is not the case that the netting batch A has instructions
    Once I attach a payment instruction B to a netting batch A, it is the case that the payment instruction B is attached to the netting batch A and the netting batch A has instructions
    Once I attach a payment instruction B to a netting batch A, it is not the case anymore that instruction is available for B

  detach a payment instruction B from a netting batch A
    I can only detach a payment instruction B from a netting batch A if it is the case that the payment instruction B is attached to the netting batch A
    I can only detach a payment instruction B from a netting batch A if it is not the case that the batch is composed of A and it is not the case that the pre settlement is approved for A
    Once I detach a payment instruction B from a netting batch A, it is the case that instruction is available for B
    Once I detach a payment instruction B from a netting batch A, it is not the case anymore that the payment instruction B is attached to the netting batch A and it is not the case anymore that the netting batch A has instructions and it is not the case anymore that routing is assigned to the netting batch A and it is not the case anymore that the netting batch A is validated and it is not the case anymore that A has a funding hold flag and it is not the case anymore that A has an execution ready flag and it is not the case anymore that A is bound to an external service

  assign a routing rule B to a netting batch A
    I can only assign a routing rule B to a netting batch A if it is the case that the batch A is initialized and routing rule is available for B
    Once I assign a routing rule B to a netting batch A, it is the case that B is assigned to A
    Once I assign a routing rule B to a netting batch A, it is not the case anymore that routing rule is available for B

  revoke a routing rule B from a netting batch A
    I can only revoke a routing rule B from a netting batch A if it is the case that B is assigned to A
    Once I revoke a routing rule B from a netting batch A, it is the case that routing rule is available for B
    Once I revoke a routing rule B from a netting batch A, it is not the case anymore that B is assigned to A

  confirm routing assignment of a routing rule B to a netting batch A
    I can only confirm routing assignment of a routing rule B to a netting batch A if it is the case that the batch A is initialized and the netting batch A has instructions and B is assigned to A
    I can only confirm routing assignment of a routing rule B to a netting batch A if it is not the case that routing is assigned to the netting batch A
    Once I confirm routing assignment of a routing rule B to a netting batch A, it is the case that routing is assigned to the netting batch A

  apply an approval token B to a netting batch A
    I can only apply an approval token B to a netting batch A if it is the case that the batch A is initialized and the netting batch A has instructions and approval token is available for B
    I can only apply an approval token B to a netting batch A if it is not the case that routing is assigned to the netting batch A
    Once I apply an approval token B to a netting batch A, it is the case that routing is assigned to the netting batch A and A has a funding hold flag
    Once I apply an approval token B to a netting batch A, it is not the case anymore that approval token is available for B

  apply clearing agent validation using a routing rule B and a clearing agent C to a netting batch A
    I can only apply clearing agent validation using a routing rule B and a clearing agent C to a netting batch A if it is the case that routing is assigned to the netting batch A and the netting batch A has instructions and B is assigned to A and clearing agent is available for C
    I can only apply clearing agent validation using a routing rule B and a clearing agent C to a netting batch A if it is not the case that the netting batch A is validated
    Once I apply clearing agent validation using a routing rule B and a clearing agent C to a netting batch A, it is the case that the netting batch A is validated
    Once I apply clearing agent validation using a routing rule B and a clearing agent C to a netting batch A, it is not the case anymore that A has a funding hold flag

  release a netting batch A via an execution window B
    I can only release a netting batch A via an execution window B if it is the case that the netting batch A has instructions and A is a secondary window confirmed for B
    I can only release a netting batch A via an execution window B if it is not the case that the netting batch A is validated
    Once I release a netting batch A via an execution window B, it is the case that the netting batch A is validated
    Once I release a netting batch A via an execution window B, it is not the case anymore that A has a funding hold flag

  bind a sending account B to a netting batch A

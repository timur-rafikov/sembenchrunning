--- TYPE HIERARCHY ---

  Everything that is a entity is also a object
  Everything that is a payout case or a funding source account or a settlement rail or a beneficiary account or a internal ledger account or a external liquidity pool or a payment instruction or a third party receipt or a execution profile or a compliance credential or a clearing reference or a alternative rail option or a operational policy is also a entity
  Everything that is a retry policy or a settlement policy is also a operational policy
  Everything that is a automated case variant or a operator case variant is also a payout case

--- ACTIONS ---

  open payout recovery case for a payout case A
    I can only open payout recovery case for a payout case A if it is not the case that A is open and it is not the case that A is finalized
    Once I open payout recovery case for a payout case A, it is the case that A is open

  attach a funding source account B to a payout case A
    I can only attach a funding source account B to a payout case A if it is the case that A is open and B is available as a funding source and A is compatible with B
    I can only attach a funding source account B to a payout case A if it is not the case that A has funding attached
    Once I attach a funding source account B to a payout case A, it is the case that A has a funding reservation from B and A has funding attached
    Once I attach a funding source account B to a payout case A, it is not the case anymore that B is available as a funding source

  release funding from a payout case A using a funding source account B
    I can only release funding from a payout case A using a funding source account B if it is the case that A has a funding reservation from B
    I can only release funding from a payout case A using a funding source account B if it is not the case that A has an execution planned and it is not the case that A has been granted approval
    Once I release funding from a payout case A using a funding source account B, it is the case that B is available as a funding source
    Once I release funding from a payout case A using a funding source account B, it is not the case anymore that A has a funding reservation from B and it is not the case anymore that A has funding attached and it is not the case anymore that A has an authorized instruction and it is not the case anymore that A has a cleared instruction and it is not the case anymore that the third party receipt for A is recorded and it is not the case anymore that the execution for A is in progress and it is not the case anymore that A requires reconciliation

  reserve a payment instruction B for a payout case A
    I can only reserve a payment instruction B for a payout case A if it is the case that A is open and B is available
    Once I reserve a payment instruction B for a payout case A, it is the case that A has a reserved instruction B
    Once I reserve a payment instruction B for a payout case A, it is not the case anymore that B is available

  release a payment instruction B from a payout case A
    I can only release a payment instruction B from a payout case A if it is the case that A has a reserved instruction B
    Once I release a payment instruction B from a payout case A, it is the case that B is available
    Once I release a payment instruction B from a payout case A, it is not the case anymore that A has a reserved instruction B

  authorize a payment instruction B for a payout case A
    I can only authorize a payment instruction B for a payout case A if it is the case that A is open and A has funding attached and A has a reserved instruction B
    I can only authorize a payment instruction B for a payout case A if it is not the case that A has an authorized instruction
    Once I authorize a payment instruction B for a payout case A, it is the case that A has an authorized instruction

  authorize instruction for a payout case A using a third party receipt B

    Once I authorize instruction for a payout case A using a third party receipt B, it is the case that A has an authorized instruction and the third party receipt for A is recorded
    Once I authorize instruction for a payout case A using a third party receipt B, it is not the case anymore that B is available

  confirm instruction for a payout case A with a payment instruction B using a compliance credential C
    I can only confirm instruction for a payout case A with a payment instruction B using a compliance credential C if it is the case that A has an authorized instruction and A has funding attached and A has a reserved instruction B and C is available
    I can only confirm instruction for a payout case A with a payment instruction B using a compliance credential C if it is not the case that A has a cleared instruction
    Once I confirm instruction for a payout case A with a payment instruction B using a compliance credential C, it is the case that A has a cleared instruction
    Once I confirm instruction for a payout case A with a payment instruction B using a compliance credential C, it is not the case anymore that the third party receipt for A is recorded

  confirm instruction for a payout case A via an execution profile B
    I can only confirm instruction for a payout case A via an execution profile B if it is the case that A has funding attached and A has a case bound execution profile B

    Once I confirm instruction for a payout case A via an execution profile B, it is the case that A has a cleared instruction
    Once I confirm instruction for a payout case A via an execution profile B, it is not the case anymore that the third party receipt for A is recorded

    Once I reserve an internal ledger account B for a payout case A, it is not the case anymore that B is available in the internal ledger

  release an internal ledger account B from a payout case A

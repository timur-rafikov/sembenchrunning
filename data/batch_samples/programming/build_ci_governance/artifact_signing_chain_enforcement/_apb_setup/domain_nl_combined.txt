--- TYPE HIERARCHY ---

  Everything that is a domain object is also a object
  Everything that is a artifact or a ci worker or a validation stage or a quality check or a test suite or a security scan or a attestation job or a policy engine or a release target or a auditor or a vuln report or a signing key or a credential is also a domain object
  Everything that is a team credential or a org credential is also a credential
  Everything that is a rc variant or a dep variant is also a artifact

--- ACTIONS ---

  register an artifact A as an artifact that is registered
    I can only register an artifact A as an artifact that is registered if it is not the case that A is an artifact that is registered and it is not the case that A is finally released
    Once I register an artifact A as an artifact that is registered, it is the case that A is an artifact that is registered

  assign a ci worker B to an artifact A as an allocated artifact
    I can only assign a ci worker B to an artifact A as an allocated artifact if it is the case that A is an artifact that is registered and B is available and B is eligible for the artifact A
    I can only assign a ci worker B to an artifact A as an allocated artifact if it is not the case that A is an allocated artifact
    Once I assign a ci worker B to an artifact A as an allocated artifact, it is the case that A is an artifact allocated to the worker B and A is an allocated artifact
    Once I assign a ci worker B to an artifact A as an allocated artifact, it is not the case anymore that B is available

  release a ci worker B from an artifact A and make the worker available
    I can only release a ci worker B from an artifact A and make the worker available if it is the case that A is an artifact allocated to the worker B
    I can only release a ci worker B from an artifact A and make the worker available if it is not the case that A is approved at the stage and it is not the case that approval is granted for A
    Once I release a ci worker B from an artifact A and make the worker available, it is the case that B is available
    Once I release a ci worker B from an artifact A and make the worker available, it is not the case anymore that A is an artifact allocated to the worker B and it is not the case anymore that A is an allocated artifact and it is not the case anymore that A is a signed artifact and it is not the case anymore that A is an attested artifact and it is not the case anymore that A requires a countersign and it is not the case anymore that the countersign for A is completed and it is not the case anymore that A has a vulnerability flag

  schedule an attestation job B for an artifact A

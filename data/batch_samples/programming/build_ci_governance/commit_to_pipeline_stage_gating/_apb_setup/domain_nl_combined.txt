DOMAIN IN NATURAL LANGUAGE

--- TYPE HIERARCHY ---

  Everything that is a change request or a executor slot or a pipeline stage or a policy or a test suite or a security scan job or a workspace slot or a reviewer or a ci tool instance or a operator credential or a infrastructure resource or a external dependency or a capacity group is also a object
  Everything that is a manual approver or a automated gate is also a capacity group
  Everything that is a pull request variant or a merge commit variant is also a change request

--- ACTIONS ---

  ingest a change request A after stage gate is passed
    I can only ingest a change request A after stage gate is passed if it is not the case that the change request A is registered and it is not the case that stage gate is passed for A
    Once I ingest a change request A after stage gate is passed, it is the case that the change request A is registered

  reserve an executor slot B for a change request A
    I can only reserve an executor slot B for a change request A if it is the case that the change request A is registered and B is available and B is compatible with A
    I can only reserve an executor slot B for a change request A if it is not the case that A has an executor allocation
    Once I reserve an executor slot B for a change request A, it is the case that the executor slot B is reserved for the change request A and A has an executor allocation
    Once I reserve an executor slot B for a change request A, it is not the case anymore that B is available

  release an executor slot B from a change request A
    I can only release an executor slot B from a change request A if it is the case that the executor slot B is reserved for the change request A
    I can only release an executor slot B from a change request A if it is not the case that the execution of the stage for A is flagged and it is not the case that approval is granted for A
    Once I release an executor slot B from a change request A, it is the case that B is available
    Once I release an executor slot B from a change request A, it is not the case anymore that the executor slot B is reserved for the change request A and it is not the case anymore that A has an executor allocation and it is not the case anymore that A is preparation ready and it is not the case anymore that the environment for A is configured and it is not the case anymore that review is assigned for A and it is not the case anymore that stage execution is confirmed for A and it is not the case anymore that a security scan was reported for A

  claim a workspace slot B for a change request A
    I can only claim a workspace slot B for a change request A if it is the case that the change request A is registered and B is available in the workspace
    Once I claim a workspace slot B for a change request A, it is the case that A is bound to the workspace slot B
    Once I claim a workspace slot B for a change request A, it is not the case anymore that B is available in the workspace

  release a workspace slot B from a change request A
    I can only release a workspace slot B from a change request A if it is the case that A is bound to the workspace slot B
    Once I release a workspace slot B from a change request A, it is the case that B is available in the workspace
    Once I release a workspace slot B from a change request A, it is not the case anymore that A is bound to the workspace slot B

  confirm workspace provision for a change request A with a workspace slot B

    I can only confirm workspace provision for a change request A with a workspace slot B if it is not the case that A is preparation ready

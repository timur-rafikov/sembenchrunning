  Everything that is a pipeline run or a source commit or a deployment target or a build configuration or a quality gate or a security scan or a cache node or a human reviewer or a promotion policy or a operator credential or a incident ticket or a cache artifact or a governance group is also a object

  Everything that is a staging variant or a production variant is also a pipeline run

--- ACTIONS ---

  initialize a pipeline run A
    I can only initialize a pipeline run A if it is not the case that A is a registered run and it is not the case that A is a promoted run
    Once I initialize a pipeline run A, it is the case that A is a registered run

  bind a source commit B to a pipeline run A
    I can only bind a source commit B to a pipeline run A if it is the case that A is a registered run and B is an available source commit and A is a run allowed by the commit B
    I can only bind a source commit B to a pipeline run A if it is not the case that the source is attached to A
    Once I bind a source commit B to a pipeline run A, it is the case that A has the source commit B and the source is attached to A
    Once I bind a source commit B to a pipeline run A, it is not the case anymore that B is an available source commit

  unbind a source commit B from a pipeline run A
    I can only unbind a source commit B from a pipeline run A if it is the case that A has the source commit B
    I can only unbind a source commit B from a pipeline run A if it is not the case that the evaluation for A passed and it is not the case that A is authorized by governance
    Once I unbind a source commit B from a pipeline run A, it is the case that B is an available source commit
    Once I unbind a source commit B from a pipeline run A, it is not the case anymore that A has the source commit B and it is not the case anymore that the source is attached to A and it is not the case anymore that local review is granted for A and it is not the case anymore that safety is verified for A and it is not the case anymore that A has a manual attention flag and it is not the case anymore that A is ready for promotion and it is not the case anymore that A requires a secondary review

  reserve a cache node B for a pipeline run A
    I can only reserve a cache node B for a pipeline run A if it is the case that A is a registered run and B is an available cache node
    Once I reserve a cache node B for a pipeline run A, it is the case that A runs the reserved cache node B
    Once I reserve a cache node B for a pipeline run A, it is not the case anymore that B is an available cache node

  release a cache node B from a pipeline run A
    I can only release a cache node B from a pipeline run A if it is the case that A runs the reserved cache node B
    Once I release a cache node B from a pipeline run A, it is the case that B is an available cache node
    Once I release a cache node B from a pipeline run A, it is not the case anymore that A runs the reserved cache node B

  grant local review for a pipeline run A using a cache node B
    I can only grant local review for a pipeline run A using a cache node B if it is the case that A is a registered run and the source is attached to A and A runs the reserved cache node B
    I can only grant local review for a pipeline run A using a cache node B if it is not the case that local review is granted for A
    Once I grant local review for a pipeline run A using a cache node B, it is the case that local review is granted for A

  grant local review for a pipeline run A by a human reviewer B
    I can only grant local review for a pipeline run A by a human reviewer B if it is the case that A is a registered run and the source is attached to A and B is an available human reviewer
    I can only grant local review for a pipeline run A by a human reviewer B if it is not the case that local review is granted for A
    Once I grant local review for a pipeline run A by a human reviewer B, it is the case that local review is granted for A and A has a manual attention flag
    Once I grant local review for a pipeline run A by a human reviewer B, it is not the case anymore that B is an available human reviewer

  operator verify cache node a cache node B for a pipeline run A using an operator credential C

    I can only operator verify cache node a cache node B for a pipeline run A using an operator credential C if it is not the case that safety is verified for A
    Once I operator verify cache node a cache node B for a pipeline run A using an operator credential C, it is the case that safety is verified for A
    Once I operator verify cache node a cache node B for a pipeline run A using an operator credential C, it is not the case anymore that A has a manual attention flag

  verify cache for a pipeline run A via a promotion policy B
    I can only verify cache for a pipeline run A via a promotion policy B if it is the case that the source is attached to A and the promotion policy B is applied to the pipeline run A
    I can only verify cache for a pipeline run A via a promotion policy B if it is not the case that safety is verified for A
    Once I verify cache for a pipeline run A via a promotion policy B, it is the case that safety is verified for A
    Once I verify cache for a pipeline run A via a promotion policy B, it is not the case anymore that A has a manual attention flag

  assign a quality gate B to a pipeline run A

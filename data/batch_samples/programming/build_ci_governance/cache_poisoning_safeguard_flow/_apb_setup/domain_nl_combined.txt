============================================================
DOMAIN IN NATURAL LANGUAGE
============================================================

--- TYPE HIERARCHY ---

  Everything that is a pipeline run or a source commit or a deployment target or a build configuration or a quality gate or a security scan or a cache node or a human reviewer or a promotion policy or a operator credential or a incident ticket or a cache artifact or a governance group is also a object
  Everything that is a approver role or a approver role alt is also a governance group
  Everything that is a staging variant or a production variant is also a pipeline run

--- ACTIONS ---

  initialize a pipeline run A
    I can only initialize a pipeline run A if it is not the case that A is a registered run and it is not the case that A is a promoted run
    Once I initialize a pipeline run A, it is the case that A is a registered run

  bind a source commit B to a pipeline run A
    I can only bind a source commit B to a pipeline run A if it is the case that A is a registered run and B is an available source commit and A is a run allowed by the commit B
    I can only bind a source commit B to a pipeline run A if it is not the case that the source is attached to A
    Once I bind a source commit B to a pipeline run A, it is the case that A has the source commit B and the source is attached to A
    Once I bind a source commit B to a pipeline run A, it is not the case anymore that B is an available source commit

  unbind a source commit B from a pipeline run A
    I can only unbind a source commit B from a pipeline run A if it is the case that A has the source commit B
    I can only unbind a source commit B from a pipeline run A if it is not the case that the evaluation for A passed and it is not the case that A is authorized by governance
    Once I unbind a source commit B from a pipeline run A, it is the case that B is an available source commit
    Once I unbind a source commit B from a pipeline run A, it is not the case anymore that A has the source commit B and it is not the case anymore that the source is attached to A and it is not the case anymore that local review is granted for A and it is not the case anymore that safety is verified for A and it is not the case anymore that A has a manual attention flag and it is not the case anymore that A is ready for promotion and it is not the case anymore that A requires a secondary review

  reserve a cache node B for a pipeline run A
    I can only reserve a cache node B for a pipeline run A if it is the case that A is a registered run and B is an available cache node
    Once I reserve a cache node B for a pipeline run A, it is the case that A runs the reserved cache node B
    Once I reserve a cache node B for a pipeline run A, it is not the case anymore that B is an available cache node

  release a cache node B from a pipeline run A
    I can only release a cache node B from a pipeline run A if it is the case that A runs the reserved cache node B
    Once I release a cache node B from a pipeline run A, it is the case that B is an available cache node
    Once I release a cache node B from a pipeline run A, it is not the case anymore that A runs the reserved cache node B

  grant local review for a pipeline run A using a cache node B
    I can only grant local review for a pipeline run A using a cache node B if it is the case that A is a registered run and the source is attached to A and A runs the reserved cache node B
    I can only grant local review for a pipeline run A using a cache node B if it is not the case that local review is granted for A
    Once I grant local review for a pipeline run A using a cache node B, it is the case that local review is granted for A

  grant local review for a pipeline run A by a human reviewer B
    I can only grant local review for a pipeline run A by a human reviewer B if it is the case that A is a registered run and the source is attached to A and B is an available human reviewer
    I can only grant local review for a pipeline run A by a human reviewer B if it is not the case that local review is granted for A
    Once I grant local review for a pipeline run A by a human reviewer B, it is the case that local review is granted for A and A has a manual attention flag
    Once I grant local review for a pipeline run A by a human reviewer B, it is not the case anymore that B is an available human reviewer

  operator verify cache node a cache node B for a pipeline run A using an operator credential C
    I can only operator verify cache node a cache node B for a pipeline run A using an operator credential C if it is the case that local review is granted for A and the source is attached to A and A runs the reserved cache node B and C is an available operator credential
    I can only operator verify cache node a cache node B for a pipeline run A using an operator credential C if it is not the case that safety is verified for A
    Once I operator verify cache node a cache node B for a pipeline run A using an operator credential C, it is the case that safety is verified for A
    Once I operator verify cache node a cache node B for a pipeline run A using an operator credential C, it is not the case anymore that A has a manual attention flag

  verify cache for a pipeline run A via a promotion policy B
    I can only verify cache for a pipeline run A via a promotion policy B if it is the case that the source is attached to A and the promotion policy B is applied to the pipeline run A
    I can only verify cache for a pipeline run A via a promotion policy B if it is not the case that safety is verified for A
    Once I verify cache for a pipeline run A via a promotion policy B, it is the case that safety is verified for A
    Once I verify cache for a pipeline run A via a promotion policy B, it is not the case anymore that A has a manual attention flag

  assign a quality gate B to a pipeline run A
    I can only assign a quality gate B to a pipeline run A if it is the case that A is a registered run and B is an available quality gate and A is a run allowed by the quality gate B
    Once I assign a quality gate B to a pipeline run A, it is the case that A is assigned to run the quality gate B
    Once I assign a quality gate B to a pipeline run A, it is not the case anymore that B is an available quality gate

  release a quality gate B from a pipeline run A
    I can only release a quality gate B from a pipeline run A if it is the case that A is assigned to run the quality gate B
    Once I release a quality gate B from a pipeline run A, it is the case that B is an available quality gate
    Once I release a quality gate B from a pipeline run A, it is not the case anymore that A is assigned to run the quality gate B

  assign a build configuration B to a pipeline run A

    Once I assign a build configuration B to a pipeline run A, it is the case that A is assigned to run the build configuration B
    Once I assign a build configuration B to a pipeline run A, it is not the case anymore that B is an available build configuration

  release a build configuration B from a pipeline run A

    Once I release a build configuration B from a pipeline run A, it is not the case anymore that A is assigned to run the build configuration B

  assign a security scan B to a pipeline run A
    I can only assign a security scan B to a pipeline run A if it is the case that A is a registered run and B is an available security scan and A is a run allowed by the security scan B
    Once I assign a security scan B to a pipeline run A, it is the case that A is assigned to run the security scan B
    Once I assign a security scan B to a pipeline run A, it is not the case anymore that B is an available security scan

  release a security scan B from a pipeline run A
    I can only release a security scan B from a pipeline run A if it is the case that A is assigned to run the security scan B
    Once I release a security scan B from a pipeline run A, it is the case that B is an available security scan
    Once I release a security scan B from a pipeline run A, it is not the case anymore that A is assigned to run the security scan B

  assign a cache artifact B to a pipeline run A
    I can only assign a cache artifact B to a pipeline run A if it is the case that A is a registered run and B is an available cache artifact and A is a run allowed by the cache artifact B
    Once I assign a cache artifact B to a pipeline run A, it is the case that A is assigned to run the cache artifact B
    Once I assign a cache artifact B to a pipeline run A, it is not the case anymore that B is an available cache artifact

  release a cache artifact B from a pipeline run A
    I can only release a cache artifact B from a pipeline run A if it is the case that A is assigned to run the cache artifact B
    Once I release a cache artifact B from a pipeline run A, it is the case that B is an available cache artifact
    Once I release a cache artifact B from a pipeline run A, it is not the case anymore that A is assigned to run the cache artifact B

  evaluate a deployment target B with a quality gate C and a build configuration D for a pipeline run A
    I can only evaluate a deployment target B with a quality gate C and a build configuration D for a pipeline run A if it is the case that the source is attached to A and A is assigned to run the quality gate C and A is assigned to run the build configuration D and B is an available deployment target and A is a run allowed by the deployment target B
    I can only evaluate a deployment target B with a quality gate C and a build configuration D for a pipeline run A if it is not the case that the evaluation for A passed
    Once I evaluate a deployment target B with a quality gate C and a build configuration D for a pipeline run A, it is the case that A is targeting the deployment B and the evaluation for A passed
    Once I evaluate a deployment target B with a quality gate C and a build configuration D for a pipeline run A, it is not the case anymore that B is an available deployment target

  evaluate a deployment target B with a security scan C and an incident ticket D for a pipeline run A
    I can only evaluate a deployment target B with a security scan C and an incident ticket D for a pipeline run A if it is the case that the source is attached to A and A is assigned to run the security scan C and D is an available incident ticket and B is an available deployment target and A is a run allowed by the deployment target B and A is a run allowed by the incident ticket D
    I can only evaluate a deployment target B with a security scan C and an incident ticket D for a pipeline run A if it is not the case that the evaluation for A passed
    Once I evaluate a deployment target B with a security scan C and an incident ticket D for a pipeline run A, it is the case that A is targeting the deployment B and the evaluation for A passed and A requires a secondary review and A has a manual attention flag
    Once I evaluate a deployment target B with a security scan C and an incident ticket D for a pipeline run A, it is not the case anymore that B is an available deployment target and it is not the case anymore that D is an available incident ticket

  clear secondary flags for a pipeline run A after evaluation with a quality gate B and a build configuration C
    I can only clear secondary flags for a pipeline run A after evaluation with a quality gate B and a build configuration C if it is the case that the evaluation for A passed and A requires a secondary review and A is assigned to run the quality gate B and A is assigned to run the build configuration C
    Once I clear secondary flags for a pipeline run A after evaluation with a quality gate B and a build configuration C, it is not the case anymore that A requires a secondary review and it is not the case anymore that A has a manual attention flag

  require governance approval for a pipeline run A with a quality gate B, a build configuration C, and an approver role D
    I can only require governance approval for a pipeline run A with a quality gate B, a build configuration C, and an approver role D if it is the case that safety is verified for A and the evaluation for A passed and A is assigned to run the quality gate B and A is assigned to run the build configuration C and A is associated with the governance group D
    I can only require governance approval for a pipeline run A with a quality gate B, a build configuration C, and an approver role D if it is not the case that A has a manual attention flag and it is not the case that A is authorized by governance
    Once I require governance approval for a pipeline run A with a quality gate B, a build configuration C, and an approver role D, it is the case that A is authorized by governance

  authorize governance for a pipeline run A with a security scan B, a cache artifact C, and an approver role alt D
    I can only authorize governance for a pipeline run A with a security scan B, a cache artifact C, and an approver role alt D if it is the case that safety is verified for A and the evaluation for A passed and A is assigned to run the security scan B and A is assigned to run the cache artifact C and A is associated with the governance group D
    I can only authorize governance for a pipeline run A with a security scan B, a cache artifact C, and an approver role alt D if it is not the case that A is authorized by governance
    Once I authorize governance for a pipeline run A with a security scan B, a cache artifact C, and an approver role alt D, it is the case that A is authorized by governance and A has a manual attention flag

  bind a promotion policy B to variant of a pipeline run A

    Once I bind a promotion policy B to variant of a pipeline run A, it is the case that A has a variant bound promotion policy B
    Once I bind a promotion policy B to variant of a pipeline run A, it is not the case anymore that B is an available promotion policy

  apply a promotion policy C between a production variant A and a staging variant B
    I can only apply a promotion policy C between a production variant A and a staging variant B if it is the case that A is a registered run and A is a production variant and A is bound to the promotion policy C and B has a variant bound promotion policy C
    I can only apply a promotion policy C between a production variant A and a staging variant B if it is not the case that the promotion policy C is applied to the pipeline run A
    Once I apply a promotion policy C between a production variant A and a staging variant B, it is the case that the promotion policy C is applied to the pipeline run A

  mark reverification as required for a pipeline run A using a cache node B and an operator credential C
    I can only mark reverification as required for a pipeline run A using a cache node B and an operator credential C if it is the case that A is a registered run and A is a production variant and safety is verified for A and A has a claimed review checkpoint and A runs the reserved cache node B and C is an available operator credential
    I can only mark reverification as required for a pipeline run A using a cache node B and an operator credential C if it is not the case that reverification is required for A
    Once I mark reverification as required for a pipeline run A using a cache node B and an operator credential C, it is the case that reverification is required for A

  finalize promotion for a pipeline run A
    I can only finalize promotion for a pipeline run A if it is the case that A is a staging variant and A is a registered run and the source is attached to A and the evaluation for A passed and A is authorized by governance and A has a claimed review checkpoint and safety is verified for A
    I can only finalize promotion for a pipeline run A if it is not the case that A is a promoted run
    Once I finalize promotion for a pipeline run A, it is the case that A is a promoted run

  finalize promotion for a pipeline run A with a promotion policy B

    I can only finalize promotion for a pipeline run A with a promotion policy B if it is not the case that A is a promoted run

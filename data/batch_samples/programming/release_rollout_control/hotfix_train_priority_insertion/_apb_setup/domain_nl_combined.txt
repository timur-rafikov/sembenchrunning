--- TYPE HIERARCHY ---

  Everything that is a deployment actor or a promotion wave or a deployment environment or a service or a artifact or a infrastructure component or a deployment target or a approval token or a release component or a runbook check or a maintenance window or a hotfix patch or a stakeholder group is also a object
  Everything that is a canary segment or a compliance constraint is also a stakeholder group
  Everything that is a release train engineer role or a release train automation agent is also a deployment actor

--- ACTIONS ---

  register release actor for a deployment actor A
    I can only register release actor for a deployment actor A if it is not the case that the actor is registered for A and it is not the case that the actor is finalized for A
    Once I register release actor for a deployment actor A, it is the case that the actor is registered for A

  assign a promotion wave B to a deployment actor A
    I can only assign a promotion wave B to a deployment actor A if it is the case that the actor is registered for A and B is available and the wave B is permitted for the actor A
    I can only assign a promotion wave B to a deployment actor A if it is not the case that the actor has an assigned wave for A
    Once I assign a promotion wave B to a deployment actor A, it is the case that the actor has wave B for A and the actor has an assigned wave for A
    Once I assign a promotion wave B to a deployment actor A, it is not the case anymore that B is available

  remove a promotion wave B from a deployment actor A
    I can only remove a promotion wave B from a deployment actor A if it is the case that the actor has wave B for A
    I can only remove a promotion wave B from a deployment actor A if it is not the case that promotion is in progress for A and it is not the case that promotion is authorized for A
    Once I remove a promotion wave B from a deployment actor A, it is the case that B is available
    Once I remove a promotion wave B from a deployment actor A, it is not the case anymore that the actor has wave B for A and it is not the case anymore that the actor has an assigned wave for A and it is not the case anymore that validation passed for A and it is not the case anymore that pre release checks completed for A and it is not the case anymore that approval is recorded for A and it is not the case anymore that canary stage is enabled for A and it is not the case anymore that a rollback is prepared for A

  reserve a deployment target B for a deployment actor A
    I can only reserve a deployment target B for a deployment actor A if it is the case that the actor is registered for A and B is available
    Once I reserve a deployment target B for a deployment actor A, it is the case that B is reserved for the actor in A
    Once I reserve a deployment target B for a deployment actor A, it is not the case anymore that B is available

  release a deployment target B from a deployment actor A
    I can only release a deployment target B from a deployment actor A if it is the case that B is reserved for the actor in A
    Once I release a deployment target B from a deployment actor A, it is the case that B is available
    Once I release a deployment target B from a deployment actor A, it is not the case anymore that B is reserved for the actor in A

  validate a deployment target B reserved for a deployment actor A

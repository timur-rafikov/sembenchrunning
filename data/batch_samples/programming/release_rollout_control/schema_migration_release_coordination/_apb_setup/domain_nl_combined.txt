DOMAIN IN NATURAL LANGUAGE

--- TYPE HIERARCHY ---

  Everything that is a promotion channel or a service instance or a deployment environment or a dependent service or a pre migration check tool or a post migration check tool or a promotion configuration or a verification agent or a schema version or a operator credential or a backup snapshot or a database cluster or a channel group is also a object
  Everything that is a canary group or a compliance profile is also a channel group
  Everything that is a channel endpoint primary or a channel endpoint secondary is also a promotion channel

--- ACTIONS ---

  register a promotion channel A as a registered channel
    I can only register a promotion channel A as a registered channel if it is not the case that A is a registered channel and it is not the case that A is finalized
    Once I register a promotion channel A as a registered channel, it is the case that A is a registered channel

  assign a promotion channel A to a service instance B
    I can only assign a promotion channel A to a service instance B if it is the case that A is a registered channel and B is available and the promotion channel A can target the service instance B
    I can only assign a promotion channel A to a service instance B if it is not the case that A has allocations
    Once I assign a promotion channel A to a service instance B, it is the case that A is assigned to the service instance B and A has allocations
    Once I assign a promotion channel A to a service instance B, it is not the case anymore that B is available

  unassign a promotion channel A from a service instance B
    I can only unassign a promotion channel A from a service instance B if it is the case that A is assigned to the service instance B
    I can only unassign a promotion channel A from a service instance B if it is not the case that A has a committed deployment and it is not the case that A is ready for rollback
    Once I unassign a promotion channel A from a service instance B, it is the case that B is available
    Once I unassign a promotion channel A from a service instance B, it is not the case anymore that A is assigned to the service instance B and it is not the case anymore that A has allocations and it is not the case anymore that A is prepared and it is not the case anymore that A is staged for migration and it is not the case anymore that A is pending verification and it is not the case anymore that A has been approved for finalization and it is not the case anymore that A has pending environment checks

  bind a promotion configuration B to a promotion channel A

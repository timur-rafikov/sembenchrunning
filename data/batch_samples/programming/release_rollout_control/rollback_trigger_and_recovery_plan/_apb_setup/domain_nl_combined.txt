--- TYPE HIERARCHY ---

  Everything that is a entity is also a object
  Everything that is a release instance or a rollout wave or a deployment target or a service dependency or a component or a infrastructure component or a runbook or a approver or a rollback template or a operator or a infrastructure resource or a configuration snapshot or a stakeholder group is also a entity
  Everything that is a dev team or a ops team is also a stakeholder group
  Everything that is a canary release variant or a full release variant is also a release instance

--- ACTIONS ---

  initialize a release instance A as a registered release
    I can only initialize a release instance A as a registered release if it is not the case that A is a registered release and it is not the case that A is finalized
    Once I initialize a release instance A as a registered release, it is the case that A is a registered release

  assign a release instance A to a rollout wave B
    I can only assign a release instance A to a rollout wave B if it is the case that A is a registered release and a wave is available for B and the release instance A is allowed in the rollout wave B
    I can only assign a release instance A to a rollout wave B if it is not the case that A is assigned
    Once I assign a release instance A to a rollout wave B, it is the case that A is assigned to the rollout wave B and A is assigned
    Once I assign a release instance A to a rollout wave B, it is not the case anymore that a wave is available for B

  unschedule a release instance A from a rollout wave B
    I can only unschedule a release instance A from a rollout wave B if it is the case that A is assigned to the rollout wave B
    I can only unschedule a release instance A from a rollout wave B if it is not the case that the deployment of A is completed and it is not the case that the promotion of A is approved
    Once I unschedule a release instance A from a rollout wave B, it is the case that a wave is available for B
    Once I unschedule a release instance A from a rollout wave B, it is not the case anymore that A is assigned to the rollout wave B and it is not the case anymore that A is assigned and it is not the case anymore that A has its release readiness validated and it is not the case anymore that operator checks are completed for A and it is not the case anymore that approval for A is recorded and it is not the case anymore that A is ready for finalization and it is not the case anymore that A is pending verification

  bind a runbook B to a release instance A
    I can only bind a runbook B to a release instance A if it is the case that A is a registered release and B is available
    Once I bind a runbook B to a release instance A, it is the case that A has a release bound runbook B
    Once I bind a runbook B to a release instance A, it is not the case anymore that B is available

  unbind a runbook B from a release instance A
    I can only unbind a runbook B from a release instance A if it is the case that A has a release bound runbook B
    Once I unbind a runbook B from a release instance A, it is the case that B is available
    Once I unbind a runbook B from a release instance A, it is not the case anymore that A has a release bound runbook B

  validate readiness of a runbook B for a release instance A
    I can only validate readiness of a runbook B for a release instance A if it is the case that A is a registered release and A is assigned and A has a release bound runbook B
    I can only validate readiness of a runbook B for a release instance A if it is not the case that A has its release readiness validated
    Once I validate readiness of a runbook B for a release instance A, it is the case that A has its release readiness validated

  apply approval for a release instance A by an approver B
    I can only apply approval for a release instance A by an approver B if it is the case that A is a registered release and A is assigned and B is available
    I can only apply approval for a release instance A by an approver B if it is not the case that A has its release readiness validated
    Once I apply approval for a release instance A by an approver B, it is the case that A has its release readiness validated and approval for A is recorded
    Once I apply approval for a release instance A by an approver B, it is not the case anymore that B is available

  perform operator precheck for a release instance A using a runbook B by an operator C
    I can only perform operator precheck for a release instance A using a runbook B by an operator C if it is the case that A has its release readiness validated and A is assigned and A has a release bound runbook B and the operator C is available
    I can only perform operator precheck for a release instance A using a runbook B by an operator C if it is not the case that operator checks are completed for A
    Once I perform operator precheck for a release instance A using a runbook B by an operator C, it is the case that operator checks are completed for A
    Once I perform operator precheck for a release instance A using a runbook B by an operator C, it is not the case anymore that approval for A is recorded

  assert that a rollback template B is available for a release instance A
    I can only assert that a rollback template B is available for a release instance A if it is the case that A is assigned and B is authorized for A
    I can only assert that a rollback template B is available for a release instance A if it is not the case that operator checks are completed for A
    Once I assert that a rollback template B is available for a release instance A, it is the case that operator checks are completed for A
    Once I assert that a rollback template B is available for a release instance A, it is not the case anymore that approval for A is recorded

  reserve a component B for a release instance A
    I can only reserve a component B for a release instance A if it is the case that A is a registered release and B is available and the release instance A is bound to the component B
    Once I reserve a component B for a release instance A, it is the case that B is reserved for A
    Once I reserve a component B for a release instance A, it is not the case anymore that B is available

  release reservation of a component B for a release instance A
    I can only release reservation of a component B for a release instance A if it is the case that B is reserved for A
    Once I release reservation of a component B for a release instance A, it is the case that B is available
    Once I release reservation of a component B for a release instance A, it is not the case anymore that B is reserved for A

  reserve a service dependency B for a release instance A

  reserve an infrastructure component B for a release instance A

    Once I reserve an infrastructure component B for a release instance A, it is not the case anymore that B is available

  release reservation of an infrastructure component B for a release instance A

    Once I release reservation of an infrastructure component B for a release instance A, it is the case that B is available
    Once I release reservation of an infrastructure component B for a release instance A, it is not the case anymore that B is reserved for A

  reserve a configuration snapshot B for a release instance A
    I can only reserve a configuration snapshot B for a release instance A if it is the case that A is a registered release and the configuration snapshot B is available and the release instance A is bound to the configuration snapshot B
    Once I reserve a configuration snapshot B for a release instance A, it is the case that B is reserved for A
    Once I reserve a configuration snapshot B for a release instance A, it is not the case anymore that the configuration snapshot B is available

  release reservation of a configuration snapshot B for a release instance A

    Once I release reservation of a configuration snapshot B for a release instance A, it is the case that the configuration snapshot B is available
    Once I release reservation of a configuration snapshot B for a release instance A, it is not the case anymore that B is reserved for A

  execute deployment of a release instance A to a deployment target B with a component C and a service dependency D
    I can only execute deployment of a release instance A to a deployment target B with a component C and a service dependency D if it is the case that A is assigned and C is reserved for A and D is reserved for A and a target is available for B and the release instance A is bound to the deployment target B
    I can only execute deployment of a release instance A to a deployment target B with a component C and a service dependency D if it is not the case that the deployment of A is completed
    Once I execute deployment of a release instance A to a deployment target B with a component C and a service dependency D, it is the case that A is deployed to B and the deployment of A is completed
    Once I execute deployment of a release instance A to a deployment target B with a component C and a service dependency D, it is not the case anymore that a target is available for B

  execute deployment of a release instance A to a deployment target B using an infrastructure component C and an infrastructure resource D
    I can only execute deployment of a release instance A to a deployment target B using an infrastructure component C and an infrastructure resource D if it is the case that A is assigned and C is reserved for A and the infrastructure resource D is available and a target is available for B and the release instance A is bound to the deployment target B and the release instance A is bound to the infrastructure resource D
    I can only execute deployment of a release instance A to a deployment target B using an infrastructure component C and an infrastructure resource D if it is not the case that the deployment of A is completed

    Once I execute deployment of a release instance A to a deployment target B using an infrastructure component C and an infrastructure resource D, it is not the case anymore that a target is available for B and it is not the case anymore that the infrastructure resource D is available

    I can only clear post deployment verification for a release instance A using a component B and a service dependency C if it is the case that the deployment of A is completed and A is pending verification and B is reserved for A and C is reserved for A
    Once I clear post deployment verification for a release instance A using a component B and a service dependency C, it is not the case anymore that A is pending verification and it is not the case anymore that approval for A is recorded

  approve promotion for a release instance A using a component B and a service dependency C with a dev team D
    I can only approve promotion for a release instance A using a component B and a service dependency C with a dev team D if it is the case that operator checks are completed for A and the deployment of A is completed and B is reserved for A and C is reserved for A and the release instance A is associated with the stakeholder group D
    I can only approve promotion for a release instance A using a component B and a service dependency C with a dev team D if it is not the case that approval for A is recorded and it is not the case that the promotion of A is approved
    Once I approve promotion for a release instance A using a component B and a service dependency C with a dev team D, it is the case that the promotion of A is approved

  ops approve promotion for a release instance A using an infrastructure component B and a configuration snapshot C with an ops team D
    I can only ops approve promotion for a release instance A using an infrastructure component B and a configuration snapshot C with an ops team D if it is the case that operator checks are completed for A and the deployment of A is completed and B is reserved for A and C is reserved for A and the release instance A is associated with the stakeholder group D
    I can only ops approve promotion for a release instance A using an infrastructure component B and a configuration snapshot C with an ops team D if it is not the case that the promotion of A is approved
    Once I ops approve promotion for a release instance A using an infrastructure component B and a configuration snapshot C with an ops team D, it is the case that the promotion of A is approved and approval for A is recorded

  finalize promotion stage for a release instance A with a component B and a service dependency C
    I can only finalize promotion stage for a release instance A with a component B and a service dependency C if it is the case that the promotion of A is approved and approval for A is recorded and B is reserved for A and C is reserved for A
    Once I finalize promotion stage for a release instance A with a component B and a service dependency C, it is the case that A is ready for finalization
    Once I finalize promotion stage for a release instance A with a component B and a service dependency C, it is not the case anymore that approval for A is recorded and it is not the case anymore that operator checks are completed for A and it is not the case anymore that A is pending verification

  mark a release instance A as linked for monitoring using a runbook B

    I can only mark a release instance A as linked for monitoring using a runbook B if it is not the case that approval for A is recorded and it is not the case that A is linked to monitoring

  mark a release instance A as linked for monitoring with an approver B

    Once I bind a rollback template B to a release instance A, it is not the case anymore that the rollback template B is available

  authorize a rollback template C for cross variant rollback from a canary release variant B to a full release variant A

    I can only authorize a rollback template C for cross variant rollback from a canary release variant B to a full release variant A if it is not the case that C is authorized for A
    Once I authorize a rollback template C for cross variant rollback from a canary release variant B to a full release variant A, it is the case that C is authorized for A

  start post deployment monitoring for a release instance A using a runbook B by an operator C

    Once I start post deployment monitoring for a release instance A using a runbook B by an operator C, it is the case that monitoring has started for A

  finalize canary release for a release instance A
    I can only finalize canary release for a release instance A if it is the case that A is a canary variant and A is a registered release and A is assigned and the deployment of A is completed and the promotion of A is approved and A is linked to monitoring and operator checks are completed for A
    I can only finalize canary release for a release instance A if it is not the case that A is finalized
    Once I finalize canary release for a release instance A, it is the case that A is finalized

  finalize full release for a release instance A using a rollback template B
    I can only finalize full release for a release instance A using a rollback template B if it is the case that A is a full variant and A is a registered release and A is assigned and the deployment of A is completed and the promotion of A is approved and A is linked to monitoring and operator checks are completed for A and B is authorized for A
    I can only finalize full release for a release instance A using a rollback template B if it is not the case that A is finalized
    Once I finalize full release for a release instance A using a rollback template B, it is the case that A is finalized

    I can only finalize release for a release instance A after monitoring has started if it is not the case that A is finalized
    Once I finalize release for a release instance A after monitoring has started, it is the case that A is finalized

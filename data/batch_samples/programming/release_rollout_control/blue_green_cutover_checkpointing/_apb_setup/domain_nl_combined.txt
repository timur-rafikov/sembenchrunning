--- TYPE HIERARCHY ---

  Everything that is a base object is also a object
  Everything that is a cutover checkpoint or a deployment slot or a verification suite or a configuration item or a dependent service or a external dependency or a network endpoint or a runbook or a traffic policy or a automation script or a backup snapshot or a canary profile or a operator group is also a base object
  Everything that is a approval gate or a operator group token is also a operator group
  Everything that is a primary cutover or a secondary cutover is also a cutover checkpoint

--- ACTIONS ---

  create a cutover checkpoint A
    I can only create a cutover checkpoint A if it is not the case that A is registered and it is not the case that A is finalized
    Once I create a cutover checkpoint A, it is the case that A is registered

  reserve a cutover checkpoint A for a deployment slot B
    I can only reserve a cutover checkpoint A for a deployment slot B if it is the case that A is registered and B is available and A is a candidate for the checkpoint slot B
    I can only reserve a cutover checkpoint A for a deployment slot B if it is not the case that A has an allocation
    Once I reserve a cutover checkpoint A for a deployment slot B, it is the case that A is allocated to deployment slot B and A has an allocation
    Once I reserve a cutover checkpoint A for a deployment slot B, it is not the case anymore that B is available

  release a cutover checkpoint A from a deployment slot B
    I can only release a cutover checkpoint A from a deployment slot B if it is the case that A is allocated to deployment slot B
    I can only release a cutover checkpoint A from a deployment slot B if it is not the case that verification for A is completed and it is not the case that validation gate for A is passed
    Once I release a cutover checkpoint A from a deployment slot B, it is the case that B is available
    Once I release a cutover checkpoint A from a deployment slot B, it is not the case anymore that A is allocated to deployment slot B and it is not the case anymore that A has an allocation and it is not the case anymore that the operator approval is recorded for A and it is not the case anymore that artifacts are staged for A and it is not the case anymore that approval for A is claimed and it is not the case anymore that A is staged for promotion and it is not the case anymore that the backup A is consumed

  attach a network endpoint B to a cutover checkpoint A
    I can only attach a network endpoint B to a cutover checkpoint A if it is the case that A is registered and B is available
    Once I attach a network endpoint B to a cutover checkpoint A, it is the case that A is bound to the network endpoint B
    Once I attach a network endpoint B to a cutover checkpoint A, it is not the case anymore that B is available

  detach a network endpoint B from a cutover checkpoint A
    I can only detach a network endpoint B from a cutover checkpoint A if it is the case that A is bound to the network endpoint B
    Once I detach a network endpoint B from a cutover checkpoint A, it is the case that B is available
    Once I detach a network endpoint B from a cutover checkpoint A, it is not the case anymore that A is bound to the network endpoint B

  configure a network endpoint B for validation with a cutover checkpoint A

--- TYPE HIERARCHY ---

  Everything that is a release or a customer tier or a deployment target or a dependency service or a canary policy or a infrastructure constraint or a cohort or a approver or a build artifact or a automation job or a integration test suite or a rollback plan or a verification artifact is also a object
  Everything that is a smoke suite or a observability check is also a verification artifact
  Everything that is a rollout unit or a control unit is also a release

  initialize a release A as a registered release
    I can only initialize a release A as a registered release if it is not the case that A is a registered release and it is not the case that release A has been finalized
    Once I initialize a release A as a registered release, it is the case that A is a registered release

  assign a release A to a customer tier B
    I can only assign a release A to a customer tier B if it is the case that A is a registered release and tier B is available and the release A is compatible with the customer tier B
    I can only assign a release A to a customer tier B if it is not the case that the release A is tier reserved
    Once I assign a release A to a customer tier B, it is the case that A is assigned to the customer tier B and the release A is tier reserved
    Once I assign a release A to a customer tier B, it is not the case anymore that tier B is available

  unassign a release A from a customer tier B
    I can only unassign a release A from a customer tier B if it is the case that A is assigned to the customer tier B
    I can only unassign a release A from a customer tier B if it is not the case that the deployment of release A is confirmed and it is not the case that the validation gate for release A has been passed
    Once I unassign a release A from a customer tier B, it is the case that tier B is available
    Once I unassign a release A from a customer tier B, it is not the case anymore that A is assigned to the customer tier B and it is not the case anymore that the release A is tier reserved and it is not the case anymore that the release A is cohort enabled and it is not the case anymore that the release A is cohort validated and it is not the case anymore that release A is pending signoff and it is not the case anymore that release A is promoted and it is not the case anymore that A requires additional review

============================================================
DOMAIN IN NATURAL LANGUAGE
============================================================

--- TYPE HIERARCHY ---

  Everything that is a change window or a approver or a time slot or a affected component or a engineer or a oncall contact or a communication channel or a escalation contact or a execution step or a runbook document or a environment or a risk assessment or a stakeholder is also a object
  Everything that is a technical stakeholder or a business stakeholder is also a stakeholder
  Everything that is a window variant a or a window variant b is also a change window


--- PREDICATES ---

  a change window exists for {?change_window}
  {?approver} is assigned as an approver for {?change_window}
  {?change_window} has an assignment state
  {?change_window} is channel confirmed
  {?change_window} is runbook validated
  {?engineer} is allocated to {?change_window}
  {?affected_component} is allocated to {?change_window}
  {?oncall_contact} is allocated to {?change_window}
  {?risk_assessment} is linked to {?change_window}
  {?change_window} is scheduled for {?time_slot}
  the schedule for {?change_window} is locked
  pre-execution checks for {?change_window} have passed
  the channel for {?change_window} is verified
  the window for {?change_window} is closed
  escalation is used for {?change_window}
  execution has started for {?change_window}
  {?change_window} has variant execution authority for {?execution_step}
  {?execution_step} is authorized in {?change_window}
  the operational readiness flag is set for {?change_window}
  {?approver} is available as an approver
  {?time_slot} is available
  {?engineer} is available
  {?affected_component} is available
  {?oncall_contact} is available for on-call duty
  {?communication_channel} is available
  {?escalation_contact} is available as an escalation contact
  {?execution_step} is open
  {?runbook_document} is present
  {?environment} is reserved
  {?risk_assessment} is available
  {?approver} is an eligible approver for the window {?change_window}
  {?change_window} is mapped to the time slot {?time_slot}
  {?change_window} is mapped to the engineer {?engineer}
  {?change_window} is mapped to the affected component {?affected_component}
  {?change_window} is mapped to the on-call contact {?oncall_contact}
  {?change_window} is mapped to the environment {?environment}
  {?change_window} is mapped to the risk assessment {?risk_assessment}
  {?change_window} is mapped to the stakeholder {?stakeholder}
  {?change_window} has a variant step relation with {?execution_step}
  {?change_window} is eligible for auto finalization
  {?change_window} requires cross variant coordination
  {?communication_channel} is reserved for the change window {?change_window}
  {?change_window} has a special execution marker
  {?change_window} has an initial variant mapping with {?execution_step}
  {?x1} is equal to {?x2}
  {?x1} is not equal to {?x2}


--- ACTIONS ---

  create a change window A
    I can only create a change window A if it is not the case that a change window exists for A and it is not the case that the window for A is closed
    Once I create a change window A, it is the case that a change window exists for A

  assign an approver B to a change window A
    I can only assign an approver B to a change window A if it is the case that a change window exists for A and B is available as an approver and B is an eligible approver for the window A
    I can only assign an approver B to a change window A if it is not the case that A has an assignment state
    Once I assign an approver B to a change window A, it is the case that B is assigned as an approver for A and A has an assignment state
    Once I assign an approver B to a change window A, it is not the case anymore that B is available as an approver

  revoke an approver B from a change window A
    I can only revoke an approver B from a change window A if it is the case that B is assigned as an approver for A
    I can only revoke an approver B from a change window A if it is not the case that the schedule for A is locked and it is not the case that pre execution checks for A have passed
    Once I revoke an approver B from a change window A, it is the case that B is available as an approver
    Once I revoke an approver B from a change window A, it is not the case anymore that B is assigned as an approver for A and it is not the case anymore that A has an assignment state and it is not the case anymore that A is channel confirmed and it is not the case anymore that A is runbook validated and it is not the case anymore that escalation is used for A and it is not the case anymore that execution has started for A and it is not the case anymore that A has a special execution marker

  reserve a communication channel B for a change window A
    I can only reserve a communication channel B for a change window A if it is the case that a change window exists for A and B is available
    Once I reserve a communication channel B for a change window A, it is the case that B is reserved for the change window A
    Once I reserve a communication channel B for a change window A, it is not the case anymore that B is available

  release a communication channel B from a change window A
    I can only release a communication channel B from a change window A if it is the case that B is reserved for the change window A
    Once I release a communication channel B from a change window A, it is the case that B is available
    Once I release a communication channel B from a change window A, it is not the case anymore that B is reserved for the change window A

  confirm a communication channel B for a change window A
    I can only confirm a communication channel B for a change window A if it is the case that a change window exists for A and A has an assignment state and B is reserved for the change window A
    I can only confirm a communication channel B for a change window A if it is not the case that A is channel confirmed
    Once I confirm a communication channel B for a change window A, it is the case that A is channel confirmed

  escalate a change window A for approval to an escalation contact B
    I can only escalate a change window A for approval to an escalation contact B if it is the case that a change window exists for A and A has an assignment state and B is available as an escalation contact
    I can only escalate a change window A for approval to an escalation contact B if it is not the case that A is channel confirmed
    Once I escalate a change window A for approval to an escalation contact B, it is the case that A is channel confirmed and escalation is used for A
    Once I escalate a change window A for approval to an escalation contact B, it is not the case anymore that B is available as an escalation contact

  validate a runbook document C for a change window A using a communication channel B
    I can only validate a runbook document C for a change window A using a communication channel B if it is the case that A is channel confirmed and A has an assignment state and B is reserved for the change window A and C is present
    I can only validate a runbook document C for a change window A using a communication channel B if it is not the case that A is runbook validated
    Once I validate a runbook document C for a change window A using a communication channel B, it is the case that A is runbook validated
    Once I validate a runbook document C for a change window A using a communication channel B, it is not the case anymore that escalation is used for A

  authorize an execution step B in a change window A for runbook validation
    I can only authorize an execution step B in a change window A for runbook validation if it is the case that A has an assignment state and B is authorized in A
    I can only authorize an execution step B in a change window A for runbook validation if it is not the case that A is runbook validated
    Once I authorize an execution step B in a change window A for runbook validation, it is the case that A is runbook validated
    Once I authorize an execution step B in a change window A for runbook validation, it is not the case anymore that escalation is used for A

  allocate an engineer B to a change window A
    I can only allocate an engineer B to a change window A if it is the case that a change window exists for A and B is available and A is mapped to the engineer B
    Once I allocate an engineer B to a change window A, it is the case that B is allocated to A
    Once I allocate an engineer B to a change window A, it is not the case anymore that B is available

  release an engineer B from a change window A
    I can only release an engineer B from a change window A if it is the case that B is allocated to A
    Once I release an engineer B from a change window A, it is the case that B is available
    Once I release an engineer B from a change window A, it is not the case anymore that B is allocated to A

  allocate an affected component B to a change window A
    I can only allocate an affected component B to a change window A if it is the case that a change window exists for A and B is available and A is mapped to the affected component B
    Once I allocate an affected component B to a change window A, it is the case that B is allocated to A
    Once I allocate an affected component B to a change window A, it is not the case anymore that B is available

  release an affected component B from a change window A
    I can only release an affected component B from a change window A if it is the case that B is allocated to A
    Once I release an affected component B from a change window A, it is the case that B is available
    Once I release an affected component B from a change window A, it is not the case anymore that B is allocated to A

  allocate an oncall contact B to a change window A
    I can only allocate an oncall contact B to a change window A if it is the case that a change window exists for A and B is available for on call duty and A is mapped to the on call contact B
    Once I allocate an oncall contact B to a change window A, it is the case that B is allocated to A
    Once I allocate an oncall contact B to a change window A, it is not the case anymore that B is available for on call duty

  release an oncall contact B from a change window A
    I can only release an oncall contact B from a change window A if it is the case that B is allocated to A
    Once I release an oncall contact B from a change window A, it is the case that B is available for on call duty
    Once I release an oncall contact B from a change window A, it is not the case anymore that B is allocated to A

  link a risk assessment B to a change window A
    I can only link a risk assessment B to a change window A if it is the case that a change window exists for A and B is available and A is mapped to the risk assessment B
    Once I link a risk assessment B to a change window A, it is the case that B is linked to A
    Once I link a risk assessment B to a change window A, it is not the case anymore that B is available

  unlink a risk assessment B from a change window A
    I can only unlink a risk assessment B from a change window A if it is the case that B is linked to A
    Once I unlink a risk assessment B from a change window A, it is the case that B is available
    Once I unlink a risk assessment B from a change window A, it is not the case anymore that B is linked to A

  schedule a change window A for a time slot B with an engineer C affecting an affected component D
    I can only schedule a change window A for a time slot B with an engineer C affecting an affected component D if it is the case that A has an assignment state and C is allocated to A and D is allocated to A and B is available and A is mapped to the time slot B
    I can only schedule a change window A for a time slot B with an engineer C affecting an affected component D if it is not the case that the schedule for A is locked
    Once I schedule a change window A for a time slot B with an engineer C affecting an affected component D, it is the case that A is scheduled for B and the schedule for A is locked
    Once I schedule a change window A for a time slot B with an engineer C affecting an affected component D, it is not the case anymore that B is available

  schedule a change window A for a time slot B with an oncall contact C in an environment D
    I can only schedule a change window A for a time slot B with an oncall contact C in an environment D if it is the case that A has an assignment state and C is allocated to A and D is reserved and B is available and A is mapped to the time slot B and A is mapped to the environment D
    I can only schedule a change window A for a time slot B with an oncall contact C in an environment D if it is not the case that the schedule for A is locked
    Once I schedule a change window A for a time slot B with an oncall contact C in an environment D, it is the case that A is scheduled for B and the schedule for A is locked and A has a special execution marker and escalation is used for A
    Once I schedule a change window A for a time slot B with an oncall contact C in an environment D, it is not the case anymore that B is available and it is not the case anymore that D is reserved

  clear special execution marker from a change window A with an engineer B for an affected component C
    I can only clear special execution marker from a change window A with an engineer B for an affected component C if it is the case that the schedule for A is locked and A has a special execution marker and B is allocated to A and C is allocated to A
    Once I clear special execution marker from a change window A with an engineer B for an affected component C, it is not the case anymore that A has a special execution marker and it is not the case anymore that escalation is used for A

  perform pre execution checks for a change window A with an engineer B, an affected component C, and a technical stakeholder D
    I can only perform pre execution checks for a change window A with an engineer B, an affected component C, and a technical stakeholder D if it is the case that A is runbook validated and the schedule for A is locked and B is allocated to A and C is allocated to A and A is mapped to the stakeholder D
    I can only perform pre execution checks for a change window A with an engineer B, an affected component C, and a technical stakeholder D if it is not the case that escalation is used for A and it is not the case that pre execution checks for A have passed
    Once I perform pre execution checks for a change window A with an engineer B, an affected component C, and a technical stakeholder D, it is the case that pre execution checks for A have passed

  perform cross domain checks for a change window A with an oncall contact B, a risk assessment C, and a business stakeholder D
    I can only perform cross domain checks for a change window A with an oncall contact B, a risk assessment C, and a business stakeholder D if it is the case that A is runbook validated and the schedule for A is locked and B is allocated to A and C is linked to A and A is mapped to the stakeholder D
    I can only perform cross domain checks for a change window A with an oncall contact B, a risk assessment C, and a business stakeholder D if it is not the case that pre execution checks for A have passed
    Once I perform cross domain checks for a change window A with an oncall contact B, a risk assessment C, and a business stakeholder D, it is the case that pre execution checks for A have passed and escalation is used for A

  mark execution as started for a change window A with an engineer B and an affected component C
    I can only mark execution as started for a change window A with an engineer B and an affected component C if it is the case that pre execution checks for A have passed and escalation is used for A and B is allocated to A and C is allocated to A
    Once I mark execution as started for a change window A with an engineer B and an affected component C, it is the case that execution has started for A
    Once I mark execution as started for a change window A with an engineer B and an affected component C, it is not the case anymore that escalation is used for A and it is not the case anymore that A is runbook validated and it is not the case anymore that A has a special execution marker

    I can only validate runbook during execution for a change window A using a communication channel B and a runbook document C if it is the case that execution has started for A and pre execution checks for A have passed and A has an assignment state and B is reserved for the change window A and C is present

  verify channel for a change window A with an escalation contact B

    Once I verify channel for a change window A with an escalation contact B, it is not the case anymore that B is available as an escalation contact

    Once I bind an execution step B to variant in a change window A, it is not the case anymore that B is open

  authorize an execution step C across variants from a window variant a window variant a B to a window variant b A
    I can only authorize an execution step C across variants from a window variant a window variant a B to a window variant b A if it is the case that a change window exists for A and A requires cross variant coordination and A has variant execution authority for C and B has a variant step relation with C
    I can only authorize an execution step C across variants from a window variant a window variant a B to a window variant b A if it is not the case that C is authorized in A
    Once I authorize an execution step C across variants from a window variant a window variant a B to a window variant b A, it is the case that C is authorized in A

  mark operational readiness for a change window A using a communication channel B and a runbook document C
    I can only mark operational readiness for a change window A using a communication channel B and a runbook document C if it is the case that a change window exists for A and A requires cross variant coordination and A is runbook validated and the channel for A is verified and B is reserved for the change window A and C is present

    Once I mark operational readiness for a change window A using a communication channel B and a runbook document C, it is the case that the operational readiness flag is set for A

  auto finalize a change window A

    I can only auto finalize a change window A if it is not the case that the window for A is closed
    Once I auto finalize a change window A, it is the case that the window for A is closed

  finalize a change window A with an execution step B authorization
    I can only finalize a change window A with an execution step B authorization if it is the case that A requires cross variant coordination and a change window exists for A and A has an assignment state and the schedule for A is locked and pre execution checks for A have passed and the channel for A is verified and A is runbook validated and B is authorized in A
    I can only finalize a change window A with an execution step B authorization if it is not the case that the window for A is closed
    Once I finalize a change window A with an execution step B authorization, it is the case that the window for A is closed

  finalize a change window A after operational readiness
    I can only finalize a change window A after operational readiness if it is the case that A requires cross variant coordination and a change window exists for A and A has an assignment state and the schedule for A is locked and pre execution checks for A have passed and the channel for A is verified and A is runbook validated and the operational readiness flag is set for A
    I can only finalize a change window A after operational readiness if it is not the case that the window for A is closed
    Once I finalize a change window A after operational readiness, it is the case that the window for A is closed

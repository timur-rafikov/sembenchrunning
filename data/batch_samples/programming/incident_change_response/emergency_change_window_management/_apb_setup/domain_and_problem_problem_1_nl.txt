--- TYPE HIERARCHY ---

  Everything that is a change window or a approver or a time slot or a affected component or a engineer or a oncall contact or a communication channel or a escalation contact or a execution step or a runbook document or a environment or a risk assessment or a stakeholder is also a object
  Everything that is a technical stakeholder or a business stakeholder is also a stakeholder
  Everything that is a window variant a or a window variant b is also a change window

--- ACTIONS ---

  create a change window A
    I can only create a change window A if it is not the case that a change window exists for A and it is not the case that the window for A is closed
    Once I create a change window A, it is the case that a change window exists for A

  assign an approver B to a change window A
    I can only assign an approver B to a change window A if it is the case that a change window exists for A and B is available as an approver and B is an eligible approver for the window A
    I can only assign an approver B to a change window A if it is not the case that A has an assignment state
    Once I assign an approver B to a change window A, it is the case that B is assigned as an approver for A and A has an assignment state
    Once I assign an approver B to a change window A, it is not the case anymore that B is available as an approver

  revoke an approver B from a change window A
    I can only revoke an approver B from a change window A if it is the case that B is assigned as an approver for A
    I can only revoke an approver B from a change window A if it is not the case that the schedule for A is locked and it is not the case that pre execution checks for A have passed
    Once I revoke an approver B from a change window A, it is the case that B is available as an approver
    Once I revoke an approver B from a change window A, it is not the case anymore that B is assigned as an approver for A and it is not the case anymore that A has an assignment state and it is not the case anymore that A is channel confirmed and it is not the case anymore that A is runbook validated and it is not the case anymore that escalation is used for A and it is not the case anymore that execution has started for A and it is not the case anymore that A has a special execution marker

  reserve a communication channel B for a change window A
    I can only reserve a communication channel B for a change window A if it is the case that a change window exists for A and B is available
    Once I reserve a communication channel B for a change window A, it is the case that B is reserved for the change window A
    Once I reserve a communication channel B for a change window A, it is not the case anymore that B is available

  release a communication channel B from a change window A
    I can only release a communication channel B from a change window A if it is the case that B is reserved for the change window A
    Once I release a communication channel B from a change window A, it is the case that B is available
    Once I release a communication channel B from a change window A, it is not the case anymore that B is reserved for the change window A

  confirm a communication channel B for a change window A
    I can only confirm a communication channel B for a change window A if it is the case that a change window exists for A and A has an assignment state and B is reserved for the change window A
    I can only confirm a communication channel B for a change window A if it is not the case that A is channel confirmed
    Once I confirm a communication channel B for a change window A, it is the case that A is channel confirmed

  escalate a change window A for approval to an escalation contact B
    I can only escalate a change window A for approval to an escalation contact B if it is the case that a change window exists for A and A has an assignment state and B is available as an escalation contact
    I can only escalate a change window A for approval to an escalation contact B if it is not the case that A is channel confirmed
    Once I escalate a change window A for approval to an escalation contact B, it is the case that A is channel confirmed and escalation is used for A
    Once I escalate a change window A for approval to an escalation contact B, it is not the case anymore that B is available as an escalation contact

  validate a runbook document C for a change window A using a communication channel B
    I can only validate a runbook document C for a change window A using a communication channel B if it is the case that A is channel confirmed and A has an assignment state and B is reserved for the change window A and C is present
    I can only validate a runbook document C for a change window A using a communication channel B if it is not the case that A is runbook validated
    Once I validate a runbook document C for a change window A using a communication channel B, it is the case that A is runbook validated
    Once I validate a runbook document C for a change window A using a communication channel B, it is not the case anymore that escalation is used for A

  authorize an execution step B in a change window A for runbook validation
    I can only authorize an execution step B in a change window A for runbook validation if it is the case that A has an assignment state and B is authorized in A
    I can only authorize an execution step B in a change window A for runbook validation if it is not the case that A is runbook validated
    Once I authorize an execution step B in a change window A for runbook validation, it is the case that A is runbook validated
    Once I authorize an execution step B in a change window A for runbook validation, it is not the case anymore that escalation is used for A

  allocate an engineer B to a change window A
    I can only allocate an engineer B to a change window A if it is the case that a change window exists for A and B is available and A is mapped to the engineer B
    Once I allocate an engineer B to a change window A, it is the case that B is allocated to A
    Once I allocate an engineer B to a change window A, it is not the case anymore that B is available

  release an engineer B from a change window A

    Once I release an engineer B from a change window A, it is not the case anymore that B is allocated to A

  schedule a change window A for a time slot B with an engineer C affecting an affected component D
    I can only schedule a change window A for a time slot B with an engineer C affecting an affected component D if it is the case that A has an assignment state and C is allocated to A and D is allocated to A and B is available and A is mapped to the time slot B

    Once I schedule a change window A for a time slot B with an engineer C affecting an affected component D, it is the case that A is scheduled for B and the schedule for A is locked

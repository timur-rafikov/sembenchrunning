--- TYPE HIERARCHY ---

  Everything that is a generic object is also a object
  Everything that is a backport request or a source branch candidate or a target release or a repository component or a patch commit or a backport candidate commit or a release window or a oncall approver or a reviewer or a ci job or a test environment or a risk tag or a approver group is also a generic object
  Everything that is a primary approver group or a secondary approver group is also a approver group
  Everything that is a urgent case variant or a standard case variant is also a backport request

--- ACTIONS ---

  open a backport request A
    I can only open a backport request A if it is not the case that the backport request A is open and it is not the case that the request A is completed
    Once I open a backport request A, it is the case that the backport request A is open

  propose a source branch candidate B for a backport request A
    I can only propose a source branch candidate B for a backport request A if it is the case that the backport request A is open and the source branch B is available and the request candidate B is compatible with the backport request A
    I can only propose a source branch candidate B for a backport request A if it is not the case that the backport request A has an assigned source branch
    Once I propose a source branch candidate B for a backport request A, it is the case that the backport request A is assigned to the source branch B and the backport request A has an assigned source branch
    Once I propose a source branch candidate B for a backport request A, it is not the case anymore that the source branch B is available

  revoke assignment of a source branch candidate B from a backport request A
    I can only revoke assignment of a source branch candidate B from a backport request A if it is the case that the backport request A is assigned to the source branch B
    I can only revoke assignment of a source branch candidate B from a backport request A if it is not the case that the evaluation of the backport request A is completed and it is not the case that the clearance for the backport request A is recorded
    Once I revoke assignment of a source branch candidate B from a backport request A, it is the case that the source branch B is available
    Once I revoke assignment of a source branch candidate B from a backport request A, it is not the case anymore that the backport request A is assigned to the source branch B and it is not the case anymore that the backport request A has an assigned source branch and it is not the case anymore that approval has been obtained for the backport request A and it is not the case anymore that validation succeeded for the backport request A and it is not the case anymore that A is pending additional checks and it is not the case anymore that A is ready for completion and it is not the case anymore that evaluation resources are allocated for A

  bind a backport request A to a release window B

    Once I bind a backport request A to a release window B, it is the case that the backport request A is bound to the release window B
    Once I bind a backport request A to a release window B, it is not the case anymore that the release window B is available

  unbind a backport request A from a release window B

    Once I unbind a backport request A from a release window B, it is the case that the release window B is available

--- TYPE HIERARCHY ---

  Everything that is a action item or a assignee candidate or a change request plan or a application component or a infrastructure resource or a external dependency or a runbook or a reviewer or a change ticket record or a automation tool or a test environment or a vendor contact or a approval role is also a object
  Everything that is a approver or a authority is also a approval role
  Everything that is a action item variant a or a action item variant b is also a action item

--- ACTIONS ---

  create an action item A as an open action item
    I can only create an action item A as an open action item if it is not the case that A is an open action item and it is not the case that A has been closed
    Once I create an action item A as an open action item, it is the case that A is an open action item

  assign an action item A to an assignee candidate B
    I can only assign an action item A to an assignee candidate B if it is the case that A is an open action item and B is available and B is eligible for the action item A
    I can only assign an action item A to an assignee candidate B if it is not the case that A has an assignment
    Once I assign an action item A to an assignee candidate B, it is the case that A is assigned to B and A has an assignment
    Once I assign an action item A to an assignee candidate B, it is not the case anymore that B is available

  unassign an action item A from an assignee candidate B
    I can only unassign an action item A from an assignee candidate B if it is the case that A is assigned to B
    I can only unassign an action item A from an assignee candidate B if it is not the case that implementation has started for A and it is not the case that approvals have been collected for A
    Once I unassign an action item A from an assignee candidate B, it is the case that B is available
    Once I unassign an action item A from an assignee candidate B, it is not the case anymore that A is assigned to B and it is not the case anymore that A has an assignment and it is not the case anymore that A has confirmed readiness and it is not the case anymore that A is implemented and it is not the case anymore that the review request for A is active and it is not the case anymore that A is scheduled for execution and it is not the case anymore that the implementation of A is recorded

  attach a runbook B to an action item A
    I can only attach a runbook B to an action item A if it is the case that A is an open action item and B is available
    Once I attach a runbook B to an action item A, it is the case that B is attached to A
    Once I attach a runbook B to an action item A, it is not the case anymore that B is available

  detach a runbook B from an action item A
    I can only detach a runbook B from an action item A if it is the case that B is attached to A
    Once I detach a runbook B from an action item A, it is the case that B is available
    Once I detach a runbook B from an action item A, it is not the case anymore that B is attached to A

  acknowledge the assignment of a runbook B to an action item A

    I can only acknowledge the assignment of a runbook B to an action item A if it is not the case that A has confirmed readiness
    Once I acknowledge the assignment of a runbook B to an action item A, it is the case that A has confirmed readiness

  assign a reviewer B to review an action item A

    Once I assign a reviewer B to review an action item A, it is not the case anymore that B is available

    Once I trigger execution of a change ticket record B for an action item A, it is not the case anymore that the review request for A is active

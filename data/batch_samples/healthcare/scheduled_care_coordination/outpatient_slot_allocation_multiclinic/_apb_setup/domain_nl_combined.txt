--- TYPE HIERARCHY ---

  Everything that is a appointment case or a clinic slot or a appointment time band or a procedure room or a clinician or a medical equipment or a telehealth endpoint or a support staff or a service profile or a consent document or a diagnostic slot or a lab resource or a logistics option is also a object
  Everything that is a patient transport option or a visit mode option is also a logistics option
  Everything that is a scheduled followup case or a new patient case is also a appointment case

--- ACTIONS ---

  initiate an appointment case A
    I can only initiate an appointment case A if it is not the case that A is an active case and it is not the case that the case is confirmed for A
    Once I initiate an appointment case A, it is the case that A is an active case

  reserve a clinic slot B for an appointment case A
    I can only reserve a clinic slot B for an appointment case A if it is the case that A is an active case and B is available at the clinic and the appointment case A is eligible for the clinic slot B
    I can only reserve a clinic slot B for an appointment case A if it is not the case that A has a reservation
    Once I reserve a clinic slot B for an appointment case A, it is the case that A is assigned to B and A has a reservation
    Once I reserve a clinic slot B for an appointment case A, it is not the case anymore that B is available at the clinic

  release reservation of a clinic slot B for an appointment case A
    I can only release reservation of a clinic slot B for an appointment case A if it is the case that A is assigned to B
    I can only release reservation of a clinic slot B for an appointment case A if it is not the case that a visit instance is created for A and it is not the case that preparation is verified for A
    Once I release reservation of a clinic slot B for an appointment case A, it is the case that B is available at the clinic
    Once I release reservation of a clinic slot B for an appointment case A, it is not the case anymore that A is assigned to B and it is not the case anymore that A has a reservation and it is not the case anymore that the endpoint for A is activated and it is not the case anymore that A is ready for the admin and it is not the case anymore that the support staff assigned flag is set for A and it is not the case anymore that preparation is finalized for A and it is not the case anymore that A has an auxiliary requirement flag

  claim a telehealth endpoint B for an appointment case A
    I can only claim a telehealth endpoint B for an appointment case A if it is the case that A is an active case and B is available
    Once I claim a telehealth endpoint B for an appointment case A, it is the case that A is assigned to the telehealth endpoint B
    Once I claim a telehealth endpoint B for an appointment case A, it is not the case anymore that B is available

  release a telehealth endpoint B from an appointment case A
    I can only release a telehealth endpoint B from an appointment case A if it is the case that A is assigned to the telehealth endpoint B
    Once I release a telehealth endpoint B from an appointment case A, it is the case that B is available
    Once I release a telehealth endpoint B from an appointment case A, it is not the case anymore that A is assigned to the telehealth endpoint B

  activate a telehealth endpoint B for an appointment case A

    I can only activate a telehealth endpoint B for an appointment case A if it is not the case that the endpoint for A is activated

  assign a support staff B and activate endpoint for an appointment case A

    Once I assign a support staff B and activate endpoint for an appointment case A, it is the case that the endpoint for A is activated and the support staff assigned flag is set for A
    Once I assign a support staff B and activate endpoint for an appointment case A, it is not the case anymore that B is available

  apply a consent document C to an appointment case A using a telehealth endpoint B

    Once I apply a consent document C to an appointment case A using a telehealth endpoint B, it is the case that A is ready for the admin
    Once I apply a consent document C to an appointment case A using a telehealth endpoint B, it is not the case anymore that the support staff assigned flag is set for A

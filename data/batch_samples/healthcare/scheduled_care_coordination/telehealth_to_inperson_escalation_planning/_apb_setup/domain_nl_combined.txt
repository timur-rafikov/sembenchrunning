--- TYPE HIERARCHY ---

  Everything that is a entity is also a object
  Everything that is a escalation case or a telehealth appointment slot or a inperson appointment slot or a clinical unit location or a provider or a clinical equipment or a telehealth resource endpoint or a patient contact resource or a clinical specialty or a scheduling staff or a diagnostic service slot or a specialist service or a team is also a entity
  Everything that is a coordination team or a escalation team is also a team
  Everything that is a telehealth case subtype or a inperson case subtype is also a escalation case

--- ACTIONS ---

  open an escalation case A
    I can only open an escalation case A if it is not the case that A is open and it is not the case that A is closed
    Once I open an escalation case A, it is the case that A is open

  assign a telehealth appointment slot B to an escalation case A
    I can only assign a telehealth appointment slot B to an escalation case A if it is the case that A is open and the telehealth slot B is available and B is an eligible telehealth slot for the case A
    I can only assign a telehealth appointment slot B to an escalation case A if it is not the case that a telehealth slot is allocated for A
    Once I assign a telehealth appointment slot B to an escalation case A, it is the case that A is assigned to the telehealth appointment slot B and a telehealth slot is allocated for A
    Once I assign a telehealth appointment slot B to an escalation case A, it is not the case anymore that the telehealth slot B is available

  release a telehealth appointment slot B from an escalation case A
    I can only release a telehealth appointment slot B from an escalation case A if it is the case that A is assigned to the telehealth appointment slot B
    I can only release a telehealth appointment slot B from an escalation case A if it is not the case that the in person booking for A is confirmed and it is not the case that the pre visit for A is coordinated
    Once I release a telehealth appointment slot B from an escalation case A, it is the case that the telehealth slot B is available
    Once I release a telehealth appointment slot B from an escalation case A, it is not the case anymore that A is assigned to the telehealth appointment slot B and it is not the case anymore that a telehealth slot is allocated for A and it is not the case anymore that escalation is required for A and it is not the case anymore that in person reservation is pending for A and it is not the case anymore that A is pending previsit and it is not the case anymore that A is ready for previsit and it is not the case anymore that A requires additional coordination

  bind a telehealth resource endpoint B to an escalation case A
    I can only bind a telehealth resource endpoint B to an escalation case A if it is the case that A is open and B is available for telehealth
    Once I bind a telehealth resource endpoint B to an escalation case A, it is the case that A is bound to the telehealth endpoint B
    Once I bind a telehealth resource endpoint B to an escalation case A, it is not the case anymore that B is available for telehealth

  unbind a telehealth resource endpoint B from an escalation case A
    I can only unbind a telehealth resource endpoint B from an escalation case A if it is the case that A is bound to the telehealth endpoint B
    Once I unbind a telehealth resource endpoint B from an escalation case A, it is the case that B is available for telehealth
    Once I unbind a telehealth resource endpoint B from an escalation case A, it is not the case anymore that A is bound to the telehealth endpoint B

  mark an escalation case A for escalation with a telehealth resource endpoint B
    I can only mark an escalation case A for escalation with a telehealth resource endpoint B if it is the case that A is open and a telehealth slot is allocated for A and A is bound to the telehealth endpoint B
    I can only mark an escalation case A for escalation with a telehealth resource endpoint B if it is not the case that escalation is required for A
    Once I mark an escalation case A for escalation with a telehealth resource endpoint B, it is the case that escalation is required for A

  notify a patient contact resource B and flag an escalation case A for escalation
    I can only notify a patient contact resource B and flag an escalation case A for escalation if it is the case that A is open and a telehealth slot is allocated for A and B is available for contact
    I can only notify a patient contact resource B and flag an escalation case A for escalation if it is not the case that escalation is required for A
    Once I notify a patient contact resource B and flag an escalation case A for escalation, it is the case that escalation is required for A and A is pending previsit
    Once I notify a patient contact resource B and flag an escalation case A for escalation, it is not the case anymore that B is available for contact

  confirm in person reservation for an escalation case A with a scheduling staff C using a telehealth resource endpoint B
    I can only confirm in person reservation for an escalation case A with a scheduling staff C using a telehealth resource endpoint B if it is the case that escalation is required for A and a telehealth slot is allocated for A and A is bound to the telehealth endpoint B and scheduling staff available C
    I can only confirm in person reservation for an escalation case A with a scheduling staff C using a telehealth resource endpoint B if it is not the case that in person reservation is pending for A
    Once I confirm in person reservation for an escalation case A with a scheduling staff C using a telehealth resource endpoint B, it is the case that in person reservation is pending for A
    Once I confirm in person reservation for an escalation case A with a scheduling staff C using a telehealth resource endpoint B, it is not the case anymore that A is pending previsit

  confirm in person reservation for an escalation case A by a clinical specialty B

    I can only confirm in person reservation for an escalation case A by a clinical specialty B if it is not the case that in person reservation is pending for A
    Once I confirm in person reservation for an escalation case A by a clinical specialty B, it is the case that in person reservation is pending for A
    Once I confirm in person reservation for an escalation case A by a clinical specialty B, it is not the case anymore that A is pending previsit

  reserve a provider B for an escalation case A

    Once I reserve a provider B for an escalation case A, it is the case that A is reserved for provider B
    Once I reserve a provider B for an escalation case A, it is not the case anymore that B is available

  release a provider B from an escalation case A
